# Presentation Cognitive Execution Workflow

## Purpose

This document defines the mandatory execution process
for AI-native presentation generation.

The AI system should NEVER
directly generate slides
without cognitive analysis first.

---

# Global Workflow

All presentation tasks must follow:

User Request
↓
Scenario Identification
↓
Audience Modeling
↓
Skill Routing
↓
Cognitive Objective Extraction
↓
Narrative Planning
↓
Visual Semantic Planning
↓
Slide Structure Planning
↓
Presentation Generation
↓
User Review Loop
↓
Local Revision
↓
Consistency Preservation
↓
Cognitive Validation

---

# Step 1 — Scenario Identification

Identify:
- teaching presentation
- scientific presentation
- faculty interview
- classroom lecture
- conference talk
- educational workshop
- defense presentation

Determine:
- communication purpose
- acceptable complexity
- expected audience reaction

---

# Step 2 — Audience Modeling

Analyze:
- expertise level
- cognitive tolerance
- teaching needs
- scientific expectations
- strategic evaluation pressure

Questions:
- Who is the audience?
- What can they understand?
- What do they expect?
- What should they remember?

---

# Step 3 — Skill Routing

Select:
- primary skill
- secondary skill
- supporting cognitive strategy

Use:
- teaching_skill
- academic_skill
- interview_skill

The selected skill controls:
- layout strategy
- narrative logic
- visual semantics
- explanation density

---

# Step 4 — Cognitive Objective Extraction

Each slide must define:
- one cognitive objective
- one focal message
- one dominant interpretation

Avoid:
- multiple competing objectives
- excessive visual competition
- information stacking

---

# Step 5 — Narrative Planning

Preferred structure:

Problem
→ Importance
→ Principle
→ Method
→ Evidence
→ Result
→ Conclusion

Narrative should feel:
- progressive
- coherent
- cognitively smooth

---

# Step 6 — Visual Semantic Planning

Visual elements must communicate meaning.

Examples:

green:
- ecology
- agriculture
- sustainability

blue:
- technology
- science
- remote sensing

orange/red:
- warning
- anomaly
- emphasis

Avoid:
- decorative color usage
- random visual systems

---

# Step 7 — Slide Structure Planning

Determine:
- slide type
- hierarchy
- layout logic
- reading flow
- visual emphasis

Slides should:
- guide eye movement naturally
- reduce interpretation difficulty
- strengthen comprehension

---

# Step 8 — Presentation Generation

Generate:
- editable slides
- consistent visual system
- cognitive hierarchy
- semantic layouts
- presentation logic

Avoid:
- decorative overload
- template imitation
- disconnected slides

---

# Step 9 — User Review Loop

After initial presentation generation,
the system should support
iterative user-guided refinement.

Users may provide:

- layout feedback
- density feedback
- scientific revision requests
- teaching clarity adjustments
- visual preference changes
- audience adaptation requests

The system should:

- interpret feedback cognitively
- prioritize local revision
- preserve presentation stability
- maintain narrative continuity

Avoid:

- unnecessary full regeneration
- unrelated slide modification
- style drift

---

# Step 10 — Local Revision

Revision should focus on:

- single-slide refinement
- figure optimization
- layout adjustment
- mechanism clarification
- teaching simplification
- density optimization

Use:

- revision_workflow.md
- layout_revision.md
- scientific_revision.md
- teaching_revision.md
- density_revision.md

Rule:

Local revision is preferred
unless global redesign is requested.

---

# Step 11 — Consistency Preservation

During revision maintain:

- typography consistency
- spacing consistency
- color semantics
- visual hierarchy
- audience adaptation
- narrative smoothness

Revision should improve communication
without destabilizing presentation structure.

Avoid:

- unnecessary redesign
- inconsistent styles
- disconnected slide logic

---

# Step 12 — Cognitive Validation

Before finalizing:

check:
- clarity
- readability
- audience understanding
- cognitive load
- visual hierarchy
- narrative smoothness

Question:

Can audience understand
the core message quickly?

If NO:
simplify.

---

# System Philosophy

The objective of this system is NOT:
beautiful slides.

The objective is:
cognitively optimized communication.
