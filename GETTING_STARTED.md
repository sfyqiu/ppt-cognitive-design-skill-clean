# Getting Started

This document explains how to use the PPT Cognitive Design Skill framework.

The framework is designed for:

- AI-native presentation generation
- cognitive-driven slide planning
- teaching-oriented PPT design
- academic presentation reasoning
- adaptive presentation intelligence

---

# Core Workflow

The framework operates using the following reasoning pipeline:

```text
User Requirement
    ↓
Requirement Parsing
    ↓
Scenario Reasoning
    ↓
Audience Reasoning
    ↓
Narrative Construction
    ↓
Slide Planning
    ↓
Layout Selection
    ↓
Visual Semantic Mapping
    ↓
Cognitive Optimization
    ↓
Presentation Evaluation
    ↓
Final Presentation Output
```

---

# Step 1 — Define User Requirements

The user should provide:

- presentation scenario
- audience type
- presentation duration
- presentation objective
- content domain
- technical complexity
- preferred style
- existing materials (optional)

---

# Example User Input

```text
I need a 15-minute teaching interview presentation
for undergraduate students.

My research background is agricultural engineering,
but the target department focuses on GIS and remote sensing.

I want the presentation to appear professional,
teaching-oriented,
and interdisciplinary.

I also want to reduce questioning risk.
```

---

# Step 2 — Run Requirement Parsing

The system extracts:

- hidden objectives
- audience cognition
- presentation risks
- narrative requirements
- visual complexity requirements

---

# Step 3 — Generate Presentation Strategy

The system determines:

- professionalism level
- cognitive density
- teaching intensity
- visual semantic strategy
- narrative rhythm
- layout complexity

---

# Step 4 — Generate PPT Structure

The system outputs:

```text
1. Presentation Positioning
2. Audience Analysis
3. Narrative Structure
4. Slide Sequence
5. Layout Recommendations
6. Visual Semantic Strategy
7. Cognitive Optimization Suggestions
8. Final Presentation Style
```

---

# Step 5 — Optimize Presentation

The framework evaluates:

- cognitive load
- visual hierarchy
- narrative clarity
- audience adaptation
- semantic consistency

The system continuously simplifies presentation structure.

---

# Recommended Usage

Best used for:

- teaching interviews

# Design Philosophy

The framework treats presentation design as:

- cognitive engineering
- audience communication
- narrative guidance
- semantic visual reasoning

NOT:

slide decoration.

the audience understands effortlessly.
