# Runtime Onboarding Guide

## Runtime Identity

This repository is NOT:

- a PPT template collection
- a slide beautification toolkit
- a prompt-only framework
- a one-shot presentation generator

This repository IS:

an AI-native presentation cognition runtime
for orchestrating:

- audience cognition modeling
- workflow routing
- semantic planning
- hierarchy stabilization
- runtime validation
- iterative refinement
- persistent cognition continuity

The objective is NOT:

rapid slide generation.

The objective IS:

cognitively stabilized communication orchestration.

---

# Runtime Entry Philosophy

The runtime MUST behave as:

```text
reason
↓
plan
↓
validate
↓
refine
↓
stabilize
↓
render
```

NOT:

```text

↓

↓
```

Communication quality is determined by:

- audience understanding
- hierarchy stability
- semantic consistency
- refinement continuity
- cognitive readability

NOT:

visual complexity.

# Step 9 — Cognitive Density Regulation

The runtime continuously evaluates:

- text quantity
- terminology burden
- visual overload
- evidence competition
- annotation density
- interpretation effort

If overload appears:

- simplify explanations
- split concepts
- enlarge focal visuals
- increase whitespace
- reduce secondary content

Reduction is preferred over overload.

Understanding quality
is more important
than information quantity.
