# Mechanism Layout Pattern System

## Purpose

This document defines the mechanism-centered
layout behavior system
for scientific presentation generation.

The objective is NOT:
decorative scientific diagrams.

The objective IS:
mechanism interpretation optimization.

Mechanism layouts should support:

- causal understanding
- scientific readability
- directional interpretation
- evidence integration
- cognitive clarity

Mechanism figures should function as:

scientific communication engines.

---

# Core Philosophy

Scientific presentations are NOT:
collections of data.

They are:

mechanism communication systems.

Mechanism layouts should help audiences understand:

- what happens
- why it happens
- how processes interact
- what evidence supports the mechanism

Strong mechanism layouts prioritize:

interpretability
over
visual complexity.

---

# Global Mechanism Principle

Every mechanism slide should establish:

Mechanism
↓
Evidence
↓
Interpretation
↓
Implication

The audience should rapidly identify:

- the dominant mechanism
- the causal pathway
- the scientific meaning

Avoid:

- disconnected evidence
- equal-weight visual competition
- publication screenshot dumping
- fragmented mechanism structures

---

# Mechanism Dominance Principle

Mechanisms should dominate:

visual attention.

The mechanism figure should:

- occupy the primary visual region
- anchor reading flow
- control interpretation order

Text should support:

the mechanism,
not compete with it.

Avoid:

- tiny mechanism diagrams
- text-dominated mechanism slides
- scattered evidence positioning

---

# Causal Flow Principle

Mechanism layouts should reveal:

directionality.

Preferred flow patterns:

- left → right
- top → bottom
- radial mechanism expansion
- layered process progression

The audience should immediately perceive:

causal movement.

Avoid:

- ambiguous directionality
- circular confusion
- visually tangled pathways

---

# Scientific Narrative Principle

Mechanism slides should tell:

a scientific story.

Preferred narrative structure:

Problem
↓
Mechanism
↓
Evidence
↓
Interpretation
↓
Scientific implication

Avoid:

- evidence dumping
- disconnected charts
- isolated figures without interpretation

Mechanisms should feel:

explained,
not merely displayed.

---

# Figure-Centered Composition

Mechanism figures are the narrative core.

Preferred behavior:

- central dominance
- isolated positioning
- annotation-supported interpretation
- directional visual flow

Avoid:

- embedded mini-figures
- dense publication composites
- figure fragmentation

Strong mechanism layouts preserve:

visual coherence.

---

# Annotation Principle

Annotations should guide:

scientific interpretation.

Preferred annotations:

- directional arrows
- concise mechanism labels
- process explanations
- evidence callouts

Annotations should remain:

- visually restrained
- highly readable
- locally isolated

Avoid:

- paragraph annotations
- statistical clutter
- excessive labeling density

---

# Evidence Integration Principle

Evidence should support:

mechanism understanding.

Preferred evidence behavior:

- positioned near related mechanism regions
- visually secondary to the mechanism
- selectively displayed

Avoid:

- detached evidence zones
- equal-weight evidence grids
- raw statistical overload

Evidence should answer:

“Why should this mechanism be trusted?”

---

# Layered Mechanism Principle

Complex mechanisms should use:

layered hierarchy.

Preferred structure:

Core process
↓
Secondary interaction
↓
Supporting evidence
↓
Interpretation

Avoid:

simultaneous equal-detail presentation.

Mechanisms should reveal:

progressive understanding.

---

# Simplification Principle

Mechanism slides should simplify:

scientific complexity.

Strong mechanism communication requires:

- reduction
- filtering
- visual prioritization

Avoid:

- publication-level detail density
- exhaustive process mapping
- complete pathway dumping

Understanding speed
is more important
than mechanism completeness.

---

# Scientific Whitespace Principle

Mechanism figures require:

breathing space.

Whitespace improves:

- process readability
- annotation clarity
- directional understanding
- evidence separation

Avoid:

- crowded mechanisms
- tightly packed process blocks
- annotation overlap

Scientific calmness improves:

interpretability.

---

# Directional Guidance Principle

Mechanism layouts should guide:

eye movement intentionally.

Preferred reading order:

Title
→ mechanism origin
→ process flow
→ evidence
→ interpretation
→ implication

Avoid:

- scattered process zones
- disconnected arrows
- unstable reading paths

Scientific interpretation should feel:

guided,
not exploratory.

---

# Comparison Mechanism Principle

Comparative mechanisms should establish:

clear contrast hierarchy.

Preferred structure:

Condition A
↔
Condition B
↓
Mechanism difference
↓
Scientific implication

Avoid:

- symmetric visual confusion
- crowded dual mechanisms
- equal-density comparison overload

Comparisons should highlight:

causal difference.

---

# Framework Mechanism Principle

Research framework slides should establish:

multi-layer scientific structure.

Preferred structure:

Research objective
↓
Scientific modules
↓
Mechanism relationships
↓
Expected outcomes

Frameworks should feel:

strategic,
not decorative.

Avoid:

- infographic-style frameworks
- decorative academic flowcharts
- excessive icon dependence

---

# Agricultural Mechanism Principle

Agricultural scientific mechanisms should emphasize:

- environmental interaction
- biological process
- soil–water–crop relationships
- spatial-temporal linkage

Preferred visuals:

- rhizosphere mechanisms
- water flow
- nutrient pathways
- crop response systems
- ecological feedback

Avoid:

- abstract engineering-only diagrams
- visually cold technical rendering

Agricultural mechanisms should feel:

scientifically grounded
and biologically interpretable.

---

# Remote Sensing Mechanism Principle

Remote sensing mechanisms should emphasize:

- spatial interpretation
- scale linkage
- inversion logic
- environmental response

Preferred visuals:

- scale transition
- inversion framework
- GIS process flow
- temporal comparison
- spectral relationship

Avoid:

- software screenshot dumping
- excessive GIS interface dependence
- visually dense workflow diagrams

Remote sensing layouts should prioritize:

scientific interpretation,
not software demonstration.

---

# AquaCrop / Modeling Principle

Modeling slides should establish:

input
→ parameterization
→ simulation
→ validation
→ prediction

Preferred visuals:

- process flow
- parameter hierarchy
- model interaction structure
- validation comparison

Avoid:

- formula overload
- code-like presentation
- dense parameter tables

Models should feel:

interpretable,
not computationally intimidating.

---

# Anti-AI-Slop Mechanism Rules

Strictly avoid:

- generic infographic mechanisms
- glowing AI arrows
- decorative 3D scientific effects
- startup-style process cards
- SaaS workflow aesthetics

Mechanism layouts should feel:

scientific,
editorial,
and cognitively intentional.

---

# Mechanism Stability Principle

Mechanism layout systems should remain:

- reusable
- interpretable
- scenario-adaptive
- scientifically stable

The objective is NOT:
visual novelty.

The objective IS:
high-quality scientific interpretation.

---

# Final Validation

Before finalizing:

Check:

- Is the mechanism immediately understandable?
- Is causal direction visually obvious?
- Is evidence properly subordinated?
- Are annotations readable?
- Is whitespace sufficient?
- Does the layout support scientific storytelling?
- Does the mechanism feel editorially mature?

If NO:

simplify
and
rebalance the mechanism hierarchy.

---

# Final Philosophy

Mechanism layouts are NOT:
scientific decoration.

Mechanism layouts ARE:

scientific cognition engineering.
