# Scientific Visual Language System

## Purpose

This document defines the scientific
visual language system
for AI-native presentation generation.

The objective is NOT:
decorative scientific slides.

The objective IS:
mechanism-centered scientific communication.

Scientific visual language should help audiences:

- understand mechanisms rapidly
- interpret evidence clearly
- follow scientific logic naturally
- trust the presented conclusions

Strong scientific visuals prioritize:

interpretability,
hierarchy,
and
scientific calmness.

---

# Core Philosophy

Scientific presentations are NOT:
paper screenshots.

They are:

scientific interpretation systems.

Scientific visuals should:

- guide interpretation
- simplify mechanism understanding
- clarify evidence relationships
- reduce cognitive friction

Strong scientific presentations feel:

editorial,
controlled,
and intellectually mature.

---

# Global Scientific Principle

Scientific slides should establish:

Mechanism
↓
Evidence
↓
Interpretation
↓
Scientific implication

The audience should rapidly identify:

- what mechanism matters
- what evidence supports it
- what conclusion should be remembered

Avoid:

- disconnected evidence
- equal-weight figure competition
- publication-density layouts
- data-first storytelling

---

# Mechanism-Centered Principle

Mechanisms should dominate:

scientific communication.

Preferred mechanism behavior:

- central visual positioning
- directional explanation
- annotation-supported interpretation
- layered scientific hierarchy

Avoid:

- tiny mechanism diagrams
- text-heavy scientific layouts
- evidence grids without mechanism structure

Scientific understanding should feel:

guided,
not extracted manually.

---

# Editorial Scientific Principle

Scientific slides should feel:

- restrained
- calm
- editorial
- visually mature

Avoid:

- corporate aesthetics
- startup scientific visuals
- decorative infographic systems
- excessive gradients
- glowing AI effects

Strong scientific slides resemble:

high-level academic communication.

---

# Evidence Hierarchy Principle

Scientific evidence should remain:

hierarchically organized.

Preferred hierarchy:

Core mechanism
↓
Supporting evidence
↓
Interpretation
↓
Implication

Avoid:

- equal-weight data dumping
- dense chart collections
- excessive statistical layering

Evidence should support:

scientific trust,
not visual intimidation.

---

# Figure Dominance Principle

Figures should function as:

scientific communication anchors.

Preferred figures:

- mechanism diagrams
- framework structures
- evidence-linked charts
- conceptual models
- process visualization
- spatial-temporal interpretation

Avoid:

- raw publication screenshots
- unreadable multi-panel figures
- crowded statistical dashboards

Figures should explain:

before they impress.

---

# Annotation Principle

Annotations should guide:

scientific interpretation.

Preferred annotation behavior:

- concise
- directional
- mechanism-linked
- visually restrained

Preferred annotation types:

- process arrows
- evidence callouts
- mechanism labels
- interpretive notes

Avoid:

- annotation overload
- paragraph explanations
- dense statistical notation

Annotations should clarify:

causal understanding.

---

# Scientific Whitespace Principle

Scientific slides require:

editorial breathing space.

Whitespace improves:

- mechanism readability
- evidence separation
- annotation clarity
- cognitive calmness

Avoid:

- compressed evidence layouts
- dense mechanism clustering
- edge-to-edge scientific composition

Scientific calmness improves:

credibility.

---

# Scientific Typography Principle

Typography should feel:

- stable
- restrained
- editorial
- scientifically mature

Preferred typography behavior:

- concise scientific titles
- restrained emphasis
- short interpretive text
- stable hierarchy scaling

Avoid:

- decorative fonts
- oversized aggressive titles
- manuscript-style paragraphs
- visually noisy scientific text

Typography should support:

scientific interpretation,
not dominate layouts.

---

Colors should communicate:

scientific meaning
and
hierarchy.

Preferred semantic colors:

Blue:
technology,
GIS,
modeling,
scientific systems

Green:
agriculture,
biology,
ecology,
vegetation

Orange / Red:
stress,
salinity,
warning,
key findings

Avoid:

- rainbow palettes
- decorative gradients
- excessive saturation
- random color variation

Scientific palettes should remain:

controlled
and
semantically stable.

---

# Scientific Reading Flow Principle

Scientific slides should guide:

scientific reasoning flow.

Preferred reading order:

Title
→ mechanism
→ evidence
→ interpretation
→ implication

Avoid:

- scattered reading paths
- disconnected evidence blocks
- unstable visual navigation

Scientific reading should feel:

guided,
structured,
and cognitively efficient.

---

# Scientific Comparison Principle

Comparisons should emphasize:

causal scientific difference.

Preferred comparison structures:

Treatment A
vs
Treatment B

Traditional system
vs
Optimized system

Low response
vs
High response

Comparisons should highlight:

mechanism variation,
not visual complexity.

Avoid:

- equal-density comparison overload
- crowded side-by-side charts
- excessive multi-variable comparison slides

---

# Framework Principle

Research framework slides should establish:

strategic scientific structure.

Preferred framework hierarchy:

Research objective
↓
Scientific modules
↓
Mechanism relationships
↓
Expected outputs

Frameworks should feel:

scientifically intentional,
not infographic-like.

Avoid:

- decorative academic flowcharts
- excessive icon dependency
- startup-style process graphics

---

# Agricultural Scientific Principle

Agricultural scientific visuals should emphasize:

- soil–water–crop interaction
- ecological relationships
- rhizosphere processes
- environmental feedback
- spatial-temporal linkage

Preferred visuals:

- mechanism pathways
- crop interaction diagrams
- irrigation processes
- nitrogen cycling systems
- salinity response mechanisms

Avoid:

- purely engineering abstraction
- visually cold process systems
- software-centered explanation

Agricultural science should feel:

biologically grounded
and scientifically interpretable.

---

Remote sensing visuals should emphasize:

- spatial interpretation
- inversion logic
- temporal dynamics
- scale relationships

Preferred visuals:

- spatial overlays
- temporal comparison
- inversion frameworks
- spectral relationships
- classification interpretation

Avoid:

- GIS interface screenshots
- software workflow dumping
- visually dense mapping systems

Remote sensing visuals should prioritize:

scientific interpretation.

---

# Modeling Scientific Principle

Modeling slides should establish:

Input
↓
Parameterization
↓
Simulation
↓
Validation
↓
Prediction

Preferred visuals:

- parameter flow
- process hierarchy
- validation comparison

Avoid:

- dense parameter tables

scientifically interpretable,
not computationally intimidating.

---

# Scientific Rhythm Principle

Scientific presentations should alternate between:

- mechanism slides
- evidence slides
- framework slides
- transition slides
- interpretation slides
- summary slides

Avoid:

continuous scientific pressure.

Strong scientific rhythm preserves:

cognitive breathing space.

---

# Anti-AI-Slop Scientific Rules

Strictly avoid:

- startup scientific aesthetics
- glowing AI scientific graphics
- SaaS infographic layouts
- decorative 3D science effects
- corporate dashboard visuals

Scientific presentations should feel:

human-designed,
editorial,
and academically mature.

---

# Scientific Stability Principle

Scientific visual systems should remain:

- reusable
- scientifically stable
- scenario-adaptive
- cognitively interpretable

The objective is NOT:
visual novelty.

The objective IS:
high-quality scientific communication.

---

# Final Validation

Before finalizing:

Check:

- Is the mechanism immediately visible?
- Is evidence hierarchy stable?
- Are figures interpretable?
- Is whitespace sufficient?
- Does scientific reasoning flow naturally?
- Does the slide feel editorially mature?
- Does the presentation avoid AI scientific clichés?

If NO:

simplify,
rebalance,
and reduce visual noise.

# Final Philosophy

Scientific visuals are NOT:
scientific decoration.

Scientific visuals ARE:

mechanism-centered cognitive communication systems.
