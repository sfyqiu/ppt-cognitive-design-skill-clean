# Teaching Revision Rules

## Purpose

This document defines how AI should
revise teaching-oriented presentation slides
based on user feedback.

The goal is:

improve audience understanding,
teaching clarity,
and cognitive accessibility.

NOT increase information quantity.

---

# Global Principle

Teaching revision should prioritize:

- intuitive understanding
- progressive explanation
- visual teaching support
- cognitive comfort
- learning effectiveness

Avoid:

- textbook-style information dumping
- excessive abstraction
- dense academic explanation

---

# Revision Intent Interpretation

## "students cannot understand"

Possible causes:

- excessive abstraction
- terminology overload
- weak visual explanation
- excessive information density

Recommended actions:

- simplify language
- reduce terminology
- add diagrams
- add comparisons
- strengthen progressive explanation

---

## "too much text"

Possible causes:

- paragraph-style layout
- insufficient visualization
- weak information chunking

Recommended actions:

- shorten text blocks
- convert text into visuals
- split content progressively
- increase whitespace

---

## "too boring"

Possible causes:

- weak visual engagement
- insufficient examples
- monotonous structure

Recommended actions:

- add visual anchors
- increase process visualization
- use comparison-based explanation
- improve slide rhythm

---

## "logic is difficult to follow"

Possible causes:

- unclear teaching sequence
- excessive transition jumps
- weak cognitive guidance

Recommended actions:

- simplify teaching flow
- strengthen step-by-step explanation
- improve reading order
- reduce simultaneous concepts

---

# Teaching Visualization Rules

Preferred visual forms:

- process diagrams
- annotated figures
- comparison layouts
- step-by-step explanation
- semantic arrows
- conceptual simplification

Avoid:

- dense tables
- large text paragraphs
- unexplained charts
- formula-heavy layouts

---

# Cognitive Load Optimization

Always ask:

Can students understand this quickly?

If NO:

- simplify
- reduce density
- increase visual explanation
- remove secondary information

Rule:

Understanding is more important
than completeness.

---

# Teaching Hierarchy Rules

Maintain clear hierarchy:

Title
→ Core concept
→ Visual explanation
→ Supporting details

One slide should teach:

- one concept
- one mechanism
- one takeaway

Avoid:

- multi-focus slides
- simultaneous explanations
- competing visuals

---

# Teaching Tone Control

Slides should feel:

- clear
- approachable
- structured
- visually guided

Avoid:

- intimidating scientific density
- excessive professionalism
- visually cold layouts

---

# Final Validation

Before finalizing teaching revision:

Check:

- Is the concept understandable?
- Is cognitive load appropriate?
- Is visual teaching support sufficient?
- Is the explanation progressive?
- Is the key takeaway clear?

If NO:

simplify and restructure.

---

# Revision Philosophy

Teaching revision is not
adding more information.

It is improving:

understanding,
clarity,
and learning experience.
