# Revision Rules System

## Purpose

This directory defines how AI should
interpret user feedback
and revise presentation content.

The revision system prioritizes:

- iterative collaboration
- local refinement
- cognitive alignment
- presentation stability
- communication effectiveness

NOT unnecessary regeneration.

---

# Revision Workflow

User feedback
→ revision intent recognition
→ cognitive diagnosis
→ revision strategy selection
→ local refinement
→ consistency validation

# Core Revision Principle

Revision should:

- improve communication
- preserve strong structure
- maintain consistency
- align with user intention

Avoid:

- unnecessary redesign
- unrelated modifications
- unstable visual changes

---

# Local Revision Priority

Unless explicitly requested:

prefer:

- element-level revision
- figure-level revision
- single-slide refinement

before:

- section reconstruction
- full presentation redesign

---

# Workflow Philosophy

Good presentation revision is not
continuous redesign.

It is:

precise cognitive refinement
through iterative collaboration.
