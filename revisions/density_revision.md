# Density Revision Rules

## Purpose

This document defines how AI should
analyze and revise presentation density
based on user feedback.

The goal is:

optimize cognitive load,
improve readability,
and strengthen communication efficiency.

NOT maximize information quantity.

---

# Global Principle

High information quantity
does not equal
high communication quality.

Presentation density should match:

- audience expertise
- presentation purpose
- cognitive tolerance
- visual readability

Rule:

Reduce before adding.

---

# Revision Intent Interpretation

## "too dense"

Possible causes:

- excessive text
- crowded figures
- weak whitespace
- too many simultaneous ideas
- excessive visual competition

Recommended actions:

- reduce text blocks
- simplify figures
- split content into multiple slides
- increase whitespace
- strengthen focal hierarchy

---

## "too empty"

Possible causes:

- weak visual anchor
- insufficient supporting information
- oversized spacing
- weak visual balance

Recommended actions:

- enlarge core visuals
- strengthen structure
- add supporting explanation
- improve information balance

---

## "hard to read"

Possible causes:

- unstable hierarchy
- excessive information compression
- weak reading flow
- insufficient chunking

Recommended actions:

- improve information grouping
- simplify layout
- increase spacing
- reduce simultaneous focal points

---

## "too much happening"

Possible causes:

- excessive colors
- competing visuals
- multiple mechanisms on one slide
- weak information prioritization

Recommended actions:

- reduce visual competition
- isolate core message
- simplify slide purpose
- reduce secondary elements

---

# Density Control Rules

One slide should communicate:

- one key message
- one mechanism
- one transition

Avoid:

- stacking unrelated content
- multi-topic slides
- compressed visual layouts

If necessary:

split slides.

---

# Text Density Rules

Prefer:

- short text blocks
- visual explanation
- semantic chunking
- progressive disclosure

Avoid:

- paragraph-heavy layouts
- excessive bullet lists
- long uninterrupted explanations

---

# Visual Density Rules

Maintain balance between:

- text
- figures
- whitespace
- annotations
- visual emphasis

Whitespace is functional.

It improves:

- readability
- focus
- cognitive comfort

---

# Audience-Based Density Adjustment

## Undergraduate Teaching

Prefer:

- low density
- large visuals
- simplified explanation
- slow information progression

---

## Academic Presentation

Prefer:

- moderate density
- mechanism-focused structure
- evidence hierarchy
- controlled complexity

---

## Interview Presentation

Prefer:

- concise structure
- fast readability
- stable hierarchy
- controlled information quantity

---

# Final Validation

Before finalizing density revision:

Check:

- Is cognitive load acceptable?
- Is the slide readable within seconds?
- Is the focal point clear?
- Is information balanced?
- Is unnecessary density removed?

If NO:

reduce and simplify.

---

# Revision Philosophy

Density revision is not
removing information randomly.

It is optimizing:

attention,
clarity,
and cognitive efficiency.
