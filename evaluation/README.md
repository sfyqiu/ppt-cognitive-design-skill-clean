# Presentation Evaluation System

## Purpose

This system defines how
AI-generated presentations
should be evaluated,
refined,
and stabilized.

The objective is NOT:
visual preference scoring.

The objective IS:

- cognitive quality evaluation
- communication effectiveness
- scientific readability
- strategic clarity
- rendering stability
- revision consistency

---

# System Role

The evaluation system acts as:

- cognitive quality control
- communication validation
- revision stabilization
- audience adaptation checking

The system should evaluate:

- generated presentations
- revised presentations
- iterative refinements
- local slide modifications

---

# Global Evaluation Philosophy

A strong presentation should:

- communicate clearly
- reduce cognitive friction
- guide interpretation naturally
- maintain visual coherence
- support audience understanding
- remain stable during revision

Evaluation should measure:

communication effectiveness,

NOT decoration richness.

---

# Evaluation Workflow

The presentation system should follow:

Generate
↓
Evaluate
↓
Identify weaknesses
↓
Refine
↓
Re-evaluate
↓
Stabilize

The system should support:

iterative improvement
through cognitive refinement.

# Primary Evaluation Dimensions

## 1. Cognitive Clarity

Evaluate:

- understanding speed
- focal point clarity
- reading smoothness
- information hierarchy

Questions:

- Can audience identify the main message quickly?
- Is the focal mechanism obvious?
- Is attention guided naturally?

Avoid:

- competing focal points
- cognitive overload
- fragmented layouts

---

## 2. Narrative Coherence

Evaluate:

- logical progression
- explanation smoothness
- causal continuity
- slide-to-slide consistency

Questions:

- Does the presentation feel progressive?
- Does each slide support the narrative?
- Are transitions cognitively smooth?

Avoid:

- disconnected slides
- abrupt logic jumps
- unordered explanation

---

## 3. Audience Adaptation

Evaluate whether presentation matches:

- audience expertise
- cognitive tolerance
- communication scenario
- teaching or scientific expectations

Questions:

- Is density appropriate?
- Is terminology understandable?
- Is explanation depth suitable?

Avoid:

- expert-level overload for students
- oversimplification for experts
- unstable communication tone

---

## 4. Visual Hierarchy

Evaluate:

- dominant visual clarity
- typography hierarchy
- spacing quality
- reading flow

Questions:

- Is hierarchy immediately visible?
- Is the dominant visual obvious?
- Does whitespace support interpretation?

Avoid:

- visual competition
- dense clustering
- chaotic composition

---

## 5. Semantic Consistency

Evaluate:

- color meaning consistency
- style coherence
- mechanism symbolism
- visual language stability

Questions:

- Do colors communicate meaning?
- Does visual language feel coherent?
- Are symbols used consistently?

Avoid:

- random palettes
- inconsistent icon systems
- conflicting styles

---

## 6. Rendering Stability

Evaluate:

- editability
- object separation
- grouping logic
- rendering cleanliness
- revision stability

Questions:

- Are elements editable?
- Are diagrams logically grouped?
- Does revision preserve consistency?
- Is layout structure stable?

Avoid:

- flattened exports
- screenshot-style rendering
- broken grouping systems
- unstable redesign

---

# Revision Evaluation

During iterative revision evaluate:

- local revision quality
- consistency preservation
- narrative stability
- style drift prevention
- audience adaptation preservation

Revision should improve:

clarity
without destabilizing
presentation structure.

---

# Risk Detection

The system should detect:

## Cognitive Overload Risk

Indicators:

- dense text
- excessive annotations
- crowded mechanisms
- multiple focal points

---

## Scientific Density Risk

Indicators:

- unreadable mechanism layers
- publication-level complexity
- statistical overload

---

## Strategic Communication Risk

Indicators:

- exaggerated claims
- visual aggression
- excessive complexity
- weak interpretability

---

## Revision Instability Risk

Indicators:

- inconsistent spacing
- typography drift
- broken hierarchy
- narrative inconsistency
- conflicting layouts

---

# Refinement Strategy

If evaluation is weak:

Prioritize:

1. simplification
2. hierarchy strengthening
3. spacing improvement
4. annotation reduction
5. mechanism clarification
6. revision stabilization

Avoid:

adding more visual elements.

Reduction is preferred.

---

# Final Validation Questions

Before final output:

- Can audience understand rapidly?
- Is the dominant message obvious?
- Does the style support communication?
- Is the mechanism interpretable?
- Does the presentation feel coherent?
- Does the slide feel cognitively calm?
- Is revision consistency preserved?

If NO:

refine again.

---

# Final Philosophy

Presentation evaluation is NOT:
beauty scoring.

It IS:

cognitive communication assessment
and iterative presentation stabilization.
