# Presentation Evaluation System

## Purpose

This system defines how
AI-generated presentations
should be evaluated and refined.

The objective is NOT:
visual preference scoring.

The objective IS:
- cognitive quality evaluation
- communication effectiveness
- scientific readability
- strategic clarity
- rendering quality assessment

---

# Global Evaluation Philosophy

A strong presentation should:

- communicate clearly
- reduce cognitive friction
- guide interpretation naturally
- maintain visual coherence
- support audience understanding

Evaluation should measure:
communication effectiveness,
NOT decoration richness.

---

# Evaluation Pipeline

AI presentation generation should follow:

Generate
↓
Evaluate
↓
Identify weaknesses
↓
Refine
↓
Re-evaluate

The system should support:
iterative improvement.

---

# Primary Evaluation Dimensions

## 1. Cognitive Clarity

Evaluate:
- understanding speed
- focal point clarity
- reading smoothness
- information hierarchy

Questions:
- Can audience identify the main message quickly?
- Is the focal mechanism obvious?
- Is attention guided naturally?

Avoid:
- competing focal points
- cognitive overload
- fragmented layouts

---

## 2. Narrative Coherence

Evaluate:
- logical progression
- explanation smoothness
- causal continuity
- slide-to-slide consistency

Questions:
- Does the presentation feel progressive?
- Does each slide support the narrative?
- Are transitions cognitively smooth?

Avoid:
- disconnected slides
- abrupt logic jumps
- unordered explanation

---

## 3. Scientific Readability

Evaluate:
- mechanism interpretability
- annotation clarity
- scientific hierarchy
- explanation accessibility

Questions:
- Is the mechanism understandable?
- Is scientific density controlled?
- Are relationships visually clear?

Avoid:
- scientific clutter
- unreadable figures
- excessive annotation density

---

## 4. Strategic Professionalism

Evaluate:
- academic credibility
- visual restraint
- professional calmness
- institutional appropriateness

Questions:
- Does the presentation feel mature?
- Does the audience trust the presenter?
- Is the visual style professionally controlled?

Avoid:
- visually aggressive layouts
- excessive self-promotion
- decorative scientific graphics

---

## 5. Visual Hierarchy

Evaluate:
- dominant visual clarity
- typography hierarchy
- spacing quality
- reading flow

Questions:
- Is the hierarchy immediately visible?
- Is the dominant visual obvious?
- Does whitespace support interpretation?

Avoid:
- visual competition
- dense clustering
- chaotic composition

---

## 6. Semantic Consistency

Evaluate:
- color meaning consistency
- style coherence
- mechanism symbolism
- visual language stability

Questions:
- Do colors communicate meaning?
- Does the visual language feel coherent?
- Are symbols used consistently?

Avoid:
- random palettes
- inconsistent icon systems
- conflicting styles

---

## 7. Rendering Quality

Evaluate:
- editability
- object separation
- grouping logic
- rendering cleanliness

Questions:
- Are elements editable?
- Are diagrams logically grouped?
- Is the rendering structurally clean?

Avoid:
- flattened exports
- screenshot-style rendering
- broken grouping systems

---

# Evaluation Scoring

Recommended scale:

1–3:
poor

4–6:
acceptable

7–8:
strong

9–10:
excellent

---

# Risk Detection

The system should detect:

## Cognitive Overload Risk

Indicators:
- dense text
- excessive annotations
- crowded mechanisms
- multiple focal points

---

## Scientific Density Risk

Indicators:
- unreadable mechanism layers
- publication-level complexity
- statistical overload

---

## Strategic Communication Risk

Indicators:
- exaggerated claims
- visual over-aggression
- excessive complexity
- low interpretability

---

# Refinement Strategy

If evaluation is weak:

Prioritize:
1. simplification
2. hierarchy strengthening
3. spacing improvement
4. annotation reduction
5. mechanism clarification

Avoid:
adding more visual elements.

Reduction is preferred.

---

# Final Validation Questions

Before final output:

- Can audience understand rapidly?
- Is the dominant message obvious?
- Does the style support communication?
- Is the mechanism interpretable?
- Does the presentation feel coherent?
- Does the slide feel cognitively calm?

If NO:
refine again.

---

# Final Philosophy

Presentation evaluation is NOT:
beauty scoring.

It IS:
cognitive communication assessment.
