# Teaching Evaluation Rules

## Purpose

This document defines how AI should
evaluate teaching-oriented presentations.

The goal is:

ensure teaching presentations are:

- understandable
- cognitively accessible
- visually guided
- progressively explained
- teaching-effective

NOT academically overloaded.

---

# Global Principle

Teaching presentation quality
is determined by:

how easily audience can understand.

NOT how much information is displayed.

A strong teaching presentation should help students:

- follow naturally
- understand quickly
- remember key concepts
- reduce interpretation difficulty

---

# Teaching Clarity Evaluation

Check:

- Is the core concept obvious?
- Is explanation intuitive?
- Is terminology understandable?
- Is the takeaway visible?
- Is information progression smooth?

Avoid:

- abstract explanation overload
- dense terminology
- simultaneous multiple concepts

---

# Cognitive Load Evaluation

Check:

- Is density appropriate for students?
- Is information chunked properly?
- Is text quantity controlled?
- Is visual explanation sufficient?

If overloaded:

- simplify explanation
- split slides
- enlarge visuals
- reduce secondary information

Reduction is preferred over overload.

---

# Teaching Narrative Evaluation

Preferred teaching progression:

Concept
→ Principle
→ Process
→ Example
→ Result
→ Summary

Check:

- Is progression step-by-step?
- Is learning flow natural?
- Are transitions smooth?
- Is explanation progressive?

Avoid:

- abrupt logic jumps
- disconnected teaching sections
- textbook dumping

---

# Teaching Visualization Evaluation

Preferred visuals:

- diagrams
- comparisons
- annotated figures
- conceptual illustrations
- process flow
- semantic arrows

Check:

- Do visuals support understanding?
- Is visual explanation stronger than text?
- Is the focal point obvious?
- Are annotations readable?

Avoid:

- large text blocks
- dense tables
- formula-heavy layouts
- visually cold slides

---

# Reading Flow Evaluation

Preferred reading flow:

Title
→ Main visual
→ Key takeaway
→ Supporting explanation

Check:

- Is attention guided naturally?
- Is hierarchy stable?
- Is eye movement predictable?

Avoid:

- visual competition
- unstable layouts
- excessive decorative elements

---

# Audience Adaptation Evaluation

Check whether presentation matches:

- undergraduate cognition
- teaching objectives
- classroom attention span
- explanation difficulty

Undergraduate teaching should prioritize:

- clarity
- simplicity
- visualization
- progressive understanding

---

# Revision Stability Evaluation

During revision check:

- Is understanding improved?
- Is density reduced appropriately?
- Is teaching rhythm preserved?
- Is visual explanation strengthened?

Avoid:

- excessive scientific complexity
- decorative redesign
- unnecessary information increase

---

# Final Validation

Before finalizing teaching presentations:

Check:

- Can students understand quickly?
- Is explanation intuitive?
- Is cognitive load appropriate?
- Is teaching flow smooth?
- Is visual guidance sufficient?

If NO:

simplify again.

---

# Evaluation Philosophy

Teaching presentation evaluation is not
judging academic sophistication.

It is evaluating:

understanding,
clarity,
and learning effectiveness.
