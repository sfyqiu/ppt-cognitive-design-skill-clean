# Teaching Evaluation Rules

## Runtime Identity

This module is NOT:

- a teaching slide scoring checklist
- an educational decoration evaluation system
- a classroom aesthetics assessment framework
- a textbook compression rating pipeline

This module IS:

a teaching cognition evaluation runtime module
for validating:

- learning accessibility
- concept clarity
- cognitive load stability
- visual teaching guidance
- educational communication continuity

The objective is NOT:

academic sophistication evaluation.

The objective IS:

learning communication stabilization.

This module cooperates with:

- evaluation/runtime/cognition_validator.md
- evaluation/runtime/hierarchy_validator.md
- evaluation/runtime/semantic_validator.md

---

## Purpose

This document defines how AI should
evaluate teaching-oriented presentations.

The goal is:

ensure teaching presentations are:

- understandable
- cognitively accessible
- visually guided
- progressively explained
- teaching-effective

NOT academically overloaded.

Evaluation should prioritize:

understanding speed
over information quantity.

---

# Global Principle

Teaching presentation quality
is determined by:

how easily audiences can understand,

NOT how much information is displayed.

A strong teaching presentation should help students:

- follow naturally
- understand quickly
- remember key concepts
- reduce interpretation difficulty
- maintain attention stability
- feel cognitively comfortable

Teaching communication should feel:

guided,
approachable,
and cognitively calm.

# Teaching Clarity Evaluation

Check:

- Is the core concept obvious?
- Is explanation intuitive?
- Is terminology understandable?
- Is the takeaway visible?
- Is information progression smooth?
- Is the dominant concept stable?

Avoid:

- abstract explanation overload
- dense terminology
- simultaneous multiple concepts
- fragmented teaching logic
- equal-weight communication

Teaching clarity should feel:

guided
and progressively understandable.

---

# Cognitive Load Evaluation

Check:

- Is density appropriate for students?
- Is information chunked properly?
- Is text quantity controlled?
- Is visual explanation sufficient?
- Is terminology burden manageable?
- Is interpretation effort minimized?

If overloaded:

- simplify explanation
- split slides
- enlarge visuals
- reduce secondary information
- simplify annotations
- increase whitespace

Reduction is preferred over overload.

Learning accessibility
is more important
than information completeness.

---

# Teaching Narrative Evaluation

Preferred teaching progression:

```text
Concept
↓
Principle
↓
Process
↓
Example
↓
Result
↓
Summary
```

Check:

- Is progression step-by-step?
- Is learning flow natural?
- Are transitions smooth?
- Is explanation progressive?
- Does each slide support understanding?
- Is narrative cognitively stable?

Avoid:

- abrupt logic jumps
- disconnected teaching sections
- textbook dumping
- terminology-first sequencing
- fragmented explanation

Teaching storytelling should feel:

progressive,
guided,
and cognitively safe.

---

# Teaching Visualization Evaluation

Preferred visuals:

- diagrams
- comparisons
- annotated figures
- conceptual illustrations
- process flow
- semantic arrows
- simplified mechanisms

Check:

- Do visuals support understanding?
- Is visual explanation stronger than text?
- Is the focal point obvious?
- Are annotations readable?
- Is hierarchy visually stable?
- Are concepts visually isolated?

Avoid:

- large text blocks
- dense tables
- formula-heavy layouts
- visually cold slides
- infographic-heavy education systems
- decorative educational graphics

Teaching visuals should explain:

meaning,
not complexity.

---

# Reading Flow Evaluation

Preferred reading flow:

```text
Title
↓
Main Visual
↓
Key Takeaway
↓
Supporting Explanation
```

Check:

- Is attention guided naturally?
- Is hierarchy stable?
- Is eye movement predictable?
- Is visual sequencing understandable?
- Is focal transition smooth?

Avoid:

- visual competition
- unstable layouts
- excessive decorative elements
- scattered explanation zones
- dashboard-style composition

Reading flow should feel:

guided
and cognitively smooth.

---

# Audience Adaptation Evaluation

Check whether presentation matches:

- undergraduate cognition
- teaching objectives
- classroom attention span
- explanation difficulty
- learner interpretation capacity

Undergraduate teaching should prioritize:

- clarity
- simplicity
- visualization
- progressive understanding
- conceptual guidance

Avoid:

- scientific compression
- excessive terminology
- abstract mechanism overload
- intimidating educational tone

Audience adaptation should feel:

intentional
and learner-centered.

---

# Revision Stability Evaluation

During revision check:

- Is understanding improved?
- Is density reduced appropriately?
- Is teaching rhythm preserved?
- Is visual explanation strengthened?
- Is hierarchy continuity preserved?
- Is semantic consistency stable?

Avoid:

- excessive scientific complexity
- decorative redesign
- unnecessary information increase
- uncontrolled regeneration
- educational atmosphere instability

Revision should improve:

learning clarity
without destabilizing
teaching continuity.

# Final Validation

Before finalizing teaching presentations:

Check:

- Can students understand quickly?
- Is explanation intuitive?
- Is cognitive load appropriate?
- Is teaching flow smooth?
- Is visual guidance sufficient?
- Is the dominant concept obvious?
- Does the presentation avoid AI teaching clichés?
- Is workflow continuity preserved?

simplify again.
