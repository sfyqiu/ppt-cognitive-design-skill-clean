# Workflow Validator System

## Purpose

This validator evaluates:

workflow cognition continuity.

The objective is NOT:
workflow categorization.

The objective IS:
runtime cognition mode stability.

Workflows regulate:

how the runtime thinks.

---

# Core Workflow Validation Philosophy

The validator MUST evaluate:

- workflow continuity
- workflow dominance
- workflow-renderer alignment
- workflow refinement stability
- workflow audience alignment

The validator MUST prevent:

workflow fragmentation.

---

# Validation 1 — Dominant Workflow Stability

## Detect

```yaml
detect:
  - missing_primary_workflow
  - equal_weight_workflow_blending
```

One workflow must remain:

dominant.
