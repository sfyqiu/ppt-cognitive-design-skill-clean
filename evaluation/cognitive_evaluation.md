# Cognitive Evaluation Rules

## Runtime Identity

This module is NOT:

- a visual scoring checklist
- a slide beauty evaluation system
- a design rating framework
- a decoration assessment pipeline

This module IS:

a cognition evaluation runtime module
for validating:

- communication clarity
- hierarchy stability
- audience interpretability
- cognitive readability
- visual guidance effectiveness

The objective is NOT:

visual preference judgment.

The objective IS:

cognitive communication stabilization.

This module cooperates with:

- evaluation/runtime/cognition_validator.md
- evaluation/runtime/hierarchy_validator.md
- evaluation/runtime/semantic_validator.md

---

## Purpose

This document defines how AI should
evaluate presentation quality
from a cognitive communication perspective.

The goal is:

ensure presentations are:

- understandable
- readable
- cognitively efficient
- audience-adapted
- visually interpretable

NOT visually overloaded.

Evaluation should prioritize:

understanding speed
over visual complexity.

---

# Global Principle

Presentation quality is determined by:

communication effectiveness,

NOT information quantity.

A strong presentation should help audiences:

- understand quickly
- follow naturally
- remember key ideas
- interpret visuals easily
- maintain cognitive stability
- reduce interpretation effort

Evaluation should prioritize:

clarity,
guidance,
and cognitive calmness.

---

Traditional evaluation systems behave as:


This runtime behaves as:

```text
evaluate

detect cognition instability
↓
diagnose communication failure
↓

↓
revalidate
↓
stabilize
```

This module continuously regulates:

- cognitive clarity

Evaluation is:

cognitive communication assessment,

---

# Core Evaluation Questions

AI should always evaluate:

- Can audience understand this quickly?
- Is the focal message clear?
- Is cognitive load acceptable?
- Is reading flow natural?
- Is visual hierarchy stable?
- Is interpretation effort minimized?
- Is communication progression smooth?
- Is audience cognition respected?

If NO:

simplify
and restructure.

---

# Cognitive Clarity Evaluation

Check:

- Is slide purpose obvious?
- Is one core message dominant?
- Is unnecessary information removed?
- Is explanation progressive?
- Is the key takeaway visible?
- Is concept transition understandable?

Avoid:

- multi-focus slides
- competing visual anchors
- excessive simultaneous concepts
- fragmented explanations
- equal-weight communication

Clarity should feel:

guided
and inevitable.

---

# Reading Flow Evaluation

Preferred reading flow:

```text
Title
↓
Core Visual
↓
Key Takeaway
↓
Supporting Details
```

Check:

- Is eye movement natural?
- Is hierarchy predictable?
- Is information sequencing smooth?
- Is attention guided intentionally?
- Are transitions cognitively stable?

Avoid:

- chaotic layouts
- disconnected sections
- unstable spacing systems
- scattered focal points
- abrupt hierarchy jumps

Reading flow should feel:

stable
and cognitively smooth.

---

# Cognitive Density Evaluation

Check:

- Is information density appropriate?
- Is text quantity controlled?
- Is whitespace sufficient?
- Are figures readable quickly?
- Is terminology burden manageable?
- Is annotation quantity controlled?

If overloaded:

- reduce text
- simplify visuals
- split slides
- remove secondary information
- enlarge focal structures
- increase whitespace

Reduction is preferred over overload.

Understanding quality
is more important
than information completeness.

---

# Audience Adaptation Evaluation

Check whether presentation matches:

- audience expertise
- cognitive tolerance
- presentation scenario
- communication purpose
- interpretation expectations

Examples:

---

## Undergraduates

Prefer:

- simpler explanation
- lower density
- stronger visualization
- progressive communication
- conceptual guidance

Avoid:

- terminology overload
- scientific compression
- abstract mechanisms

---

## Experts

Prefer:

- concise logic
- stronger evidence
- controlled density
- mechanism-centered hierarchy

Avoid:

- oversimplification
- decorative storytelling
- infographic-heavy science

---

## Interview Committees

Prefer:

- strategic clarity
- balanced professionalism
- readable structure
- restrained communication

Avoid:

- aggressive hierarchy
- publication dumping
- startup visual behavior

Audience adaptation should feel:

intentional
and cognitively aware.

---

# Visual Hierarchy Evaluation

Check:

- Is focal emphasis clear?
- Are important elements dominant?
- Is visual balance stable?
- Is spacing controlled?
- Is attention guidance obvious?
- Is hierarchy progression smooth?

Avoid:

- equal-weight elements
- visual competition
- decorative overload
- crowded layouts
- dashboard-style composition

Hierarchy should guide:

attention
before interpretation.

---

# Narrative Evaluation

Check:

- Is presentation progression smooth?
- Is logic coherent?
- Are transitions understandable?
- Is the narrative cognitively stable?
- Does each slide support the narrative?
- Is communication progression interpretable?

Preferred narrative feeling:

- progressive
- inevitable
- interpretable
- coherent
- strategically guided

Avoid:

- abrupt jumps
- disconnected mechanisms
- isolated results
- fragmented storytelling
- disconnected sections

Narrative should feel:

human-designed
and cognitively intentional.

---

# Revision Stability Evaluation

During iterative revision check:

- Is consistency preserved?
- Is audience adaptation stable?
- Is hierarchy maintained?
- Is style drift avoided?
- Is communication continuity preserved?
- Is semantic consistency stable?

Local revision should improve:

clarity
without destabilizing
presentation structure.

Avoid:

- uncontrolled redesign
- random regeneration
- decorative escalation
- inconsistent refinement behavior
