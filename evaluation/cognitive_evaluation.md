# Cognitive Evaluation Rules

## Purpose

This document defines how AI should
evaluate presentation quality
from a cognitive communication perspective.

The goal is:

ensure presentations are:

- understandable
- readable
- cognitively efficient
- audience-adapted
- visually interpretable

NOT visually overloaded.

---

# Global Principle

Presentation quality is determined by:

communication effectiveness,

NOT information quantity.

A strong presentation should help audience:

- understand quickly
- follow naturally
- remember key ideas
- interpret visuals easily

---

# Core Evaluation Questions

AI should always evaluate:

- Can audience understand this quickly?
- Is the focal message clear?
- Is cognitive load acceptable?
- Is reading flow natural?
- Is visual hierarchy stable?
- Is interpretation effort minimized?

If NO:

simplify and restructure.

---

# Cognitive Clarity Evaluation

Check:

- Is slide purpose obvious?
- Is one core message dominant?
- Is unnecessary information removed?
- Is explanation progressive?
- Is the key takeaway visible?

Avoid:

- multi-focus slides
- competing visual anchors
- excessive simultaneous concepts

---

# Reading Flow Evaluation

Preferred reading flow:

Title
→ Core visual
→ Key takeaway
→ Supporting details

Check:

- Is eye movement natural?
- Is hierarchy predictable?
- Is information sequencing smooth?

Avoid:

- chaotic layouts
- disconnected sections
- unstable spacing systems

---

# Cognitive Density Evaluation

Check:

- Is information density appropriate?
- Is text quantity controlled?
- Is whitespace sufficient?
- Are figures readable quickly?

If overloaded:

- reduce text
- simplify visuals
- split slides
- remove secondary information

Reduction is preferred over overload.

---

# Audience Adaptation Evaluation

Check whether presentation matches:

- audience expertise
- cognitive tolerance
- presentation scenario
- communication purpose

Examples:

Undergraduates:
- simpler explanation
- lower density
- stronger visualization

Experts:
- concise logic
- stronger evidence
- controlled density

Interview committees:
- strategic clarity
- balanced professionalism
- readable structure

---

# Visual Hierarchy Evaluation

Check:

- Is focal emphasis clear?
- Are important elements dominant?
- Is visual balance stable?
- Is spacing controlled?

Avoid:

- equal-weight elements
- visual competition
- decorative overload

---

# Narrative Evaluation

Check:

- Is presentation progression smooth?
- Is logic coherent?
- Are transitions understandable?
- Is the narrative cognitively stable?

Preferred narrative feeling:

- progressive
- inevitable
- interpretable

Avoid:

- abrupt jumps
- disconnected mechanisms
- isolated results

---

# Revision Stability Evaluation

During iterative revision check:

- Is consistency preserved?
- Is audience adaptation stable?
- Is hierarchy maintained?
- Is style drift avoided?

Local revision should improve:

clarity
without destabilizing
presentation structure.

---

# Final Validation

Before finalizing presentation:

Check:

- Is communication effective?
- Is understanding prioritized?
- Is interpretation easy?
- Is density controlled?
- Is visual guidance clear?

If NO:

simplify again.

---

# Evaluation Philosophy

Presentation evaluation is not
judging decoration quality.

It is evaluating:

cognitive communication effectiveness.
