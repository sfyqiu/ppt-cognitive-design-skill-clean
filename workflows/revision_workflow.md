# Presentation Revision Workflow

## Purpose

This workflow defines how an AI system should
analyze, interpret, and revise
PowerPoint presentations based on user feedback.

The workflow prioritizes:

- iterative collaboration
- cognitive alignment
- local revision stability
- visual consistency
- narrative preservation

NOT full regeneration by default.

---

# Global Principle

Presentation revision is NOT
simple visual modification.

It is:

- cognitive refinement
- communication adjustment
- audience adaptation
- presentation optimization

The goal is:

align presentation output
with user intention.

---

# Stage 1 — Revision Intent Recognition

AI should identify:

- what the user dislikes
- what the user expects
- what cognitive problem exists
- what presentation goal is missing

User feedback may be vague.

Examples:

"too messy"
→ visual hierarchy conflict
→ excessive density
→ competing focal points

"too empty"
→ weak visual anchor
→ insufficient information balance

"not academic enough"
→ weak mechanism logic
→ insufficient evidence hierarchy
→ unstable typography

"undergraduates cannot understand"
→ excessive abstraction
→ excessive terminology
→ insufficient visualization

Rule:

Interpret feedback cognitively,
not literally.

---

# Stage 2 — Revision Scope Detection

AI should determine whether revision affects:

- single element
- figure
- text block
- single slide
- section
- full presentation

Rule:

Local revision is preferred
unless global redesign is requested.

Avoid:

- unnecessary full regeneration
- unrelated slide modification
- narrative destruction

---

# Stage 3 — Cognitive Problem Diagnosis

AI should identify whether the problem relates to:

- cognitive overload
- weak hierarchy
- unclear narrative
- poor visual balance
- insufficient mechanism explanation
- audience mismatch
- weak emphasis
- inconsistent style

Revision should solve
root cognitive problems,
not only visual symptoms.

---

# Stage 4 — Revision Strategy Selection

Select appropriate revision strategy.

Examples:

## Density Reduction

Use:
- fewer text blocks
- increased whitespace
- simplified layouts
- fewer competing visuals

---

## Scientific Enhancement

Use:
- stronger mechanism chains
- clearer evidence logic
- more professional hierarchy
- cleaner scientific layout

---

## Teaching Optimization

Use:
- diagrams
- comparisons
- annotations
- simplified explanations
- progressive information flow

---

## Visual Stabilization

Use:
- consistent spacing
- stable typography
- semantic color control
- aligned visual hierarchy

---

# Stage 5 — Local Revision Execution

When revising slides:

preserve:

- original narrative logic
- existing strong elements
- slide purpose
- audience adaptation strategy

Prioritize:

- partial modification
- local optimization
- controlled refinement

Avoid:

- redesigning successful sections
- unnecessary visual complexity
- style drift

---

# Stage 6 — Consistency Preservation

During revision maintain consistency in:

- typography
- spacing
- colors
- icon style
- visual hierarchy
- narrative tone
- audience strategy

Rule:

Revision should improve coherence,
not introduce instability.

---

# Stage 7 — Iterative Refinement

Presentation revision should support:

generate
→ review
→ revise
→ refine
→ finalize

AI should continuously align with:

- user expectations
- audience cognition
- presentation goals
- communication effectiveness

Revision is iterative collaboration,
not one-time correction.

---

# Final Validation

Before finalizing revisions:

Check:

- Is the revised slide clearer?
- Is cognitive load improved?
- Is the focal point stronger?
- Is audience understanding improved?
- Is visual consistency preserved?
- Does the revision align with user intention?

If NO:

continue refinement.

---

# Workflow Philosophy

Presentation revision is:

cognitive alignment engineering.

The goal is not to redesign everything.

The goal is to precisely improve
communication effectiveness
while preserving presentation stability.
