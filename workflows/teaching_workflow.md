# Teaching PPT Design Workflow

## Purpose

This workflow defines how an AI system should
analyze, structure, and design a teaching-oriented
PowerPoint presentation.

The workflow prioritizes:
- cognitive clarity
- audience understanding
- visual storytelling
- teaching effectiveness

NOT decorative complexity.

---

# Global Principle

A teaching PPT is NOT a document.

It is:
- guided cognition
- structured explanation
- visual teaching support

The audience should:
- understand quickly
- follow naturally
- remember key ideas

---

# Stage 1 — Scenario Analysis

Identify:

- teaching interview
- classroom lecture
- training session
- academic course
- undergraduate teaching
- graduate teaching

Determine:
- presentation goal
- expected audience reaction
- acceptable complexity

Output:
- scenario type
- cognitive depth
- visual strategy

---

# Stage 2 — User Profile Analysis

Analyze:
- discipline
- professional background
- strengths
- weaknesses
- preferred communication style

Goal:
Design should amplify strengths
while reducing exposure of weaknesses.

Example:
If user lacks theoretical depth:
- reduce theoretical overload
- increase application examples
- emphasize intuitive explanation

---

# Stage 3 — Audience Modeling

Identify:
- audience expertise
- learning ability
- attention span
- cognitive tolerance

Rules:

Undergraduates:
- fewer formulas
- more diagrams
- clearer examples
- slower information density

Experts:
- more concise
- stronger logic
- stronger evidence

Interview committees:
- stronger professionalism
- stronger structure
- stronger credibility

---

# Stage 4 — Teaching Objective Extraction

For every section identify:

- what should audience understand?
- what should audience remember?
- what problem does this slide solve?

Rule:
Every slide must have:
- one cognitive objective
- one focal point
- one key takeaway

Avoid:
- multi-focus slides
- information stacking
- unnecessary details

---

# Stage 5 — Narrative Structure Design

Preferred structure:

Problem
→ Why it matters
→ Principle
→ Method
→ Example
→ Result
→ Summary

Teaching logic should feel:
- progressive
- natural
- easy to follow

Avoid:
- textbook dumping
- unordered explanation
- disconnected slides

---

# Stage 6 — Slide Classification

Classify slides into:

- cover
- intro
- principle
- comparison
- process
- flow
- result
- summary
- thank-you

Rule:
Different slide types require
different layout strategies.

---

# Stage 7 — Layout Strategy

## Cover
- large visual background
- minimal text
- strong visual center

## Intro
- problem-driven
- emotional engagement
- visual hook

## Principle
- diagram + explanation
- simplified visual teaching

## Process / Flow
- arrows
- timelines
- progressive explanation

## Result
- figure + takeaway
- highlight key insight

## Summary
- concise
- memorable
- low information density

---

# Stage 8 — Visual Semantic Mapping

Map visual language to meaning.

Examples:

green:
- agriculture
- ecology
- vegetation

blue:
- GIS
- technology
- remote sensing

orange/red:
- warning
- anomaly
- emphasis

Rule:
Colors should communicate meaning,
not decoration.

---

# Stage 9 — Cognitive Load Optimization

Always ask:

Can audience understand
within 3 seconds?

If NO:
- reduce text
- simplify layout
- reduce colors
- remove unnecessary elements

Rule:
Reduction is preferred over addition.

---

# Stage 10 — Visual Hierarchy

Always establish:

Title
↓
Subtitle
↓
Core visual
↓
Supporting information

Most important elements should be:
- larger
- brighter
- more central

Never allow all elements
to compete equally.

---

# Stage 11 — Reading Flow Design

Control audience eye movement.

Preferred reading order:

Title
→ image
→ key conclusion
→ details

Slides should guide attention naturally.

---

# Stage 12 — Teaching Adaptation

Teaching slides should:
- explain visually
- reduce abstraction
- reduce overload
- emphasize understanding

Preferred:
- diagrams
- comparisons
- annotated figures
- process visualization

Avoid:
- dense paragraphs
- excessive formulas
- academic clutter

---

# Stage 13 — Style Consistency

Maintain consistency in:
- colors
- icon style
- spacing
- typography
- visual language

Consistency builds professionalism
and cognitive comfort.

---

# Final Validation

Before finalizing presentation:

Check:
- Is the narrative smooth?
- Is cognitive load acceptable?
- Is audience understanding prioritized?
- Does the PPT reflect user strengths?
- Is visual hierarchy clear?
- Does each slide have one focal point?

If NO:
iterate and simplify.

---

# Workflow Philosophy

A good teaching PPT does not
show how much the presenter knows.

It helps audience understand
as easily as possible.

Teaching PPT design is:
cognitive experience engineering.
