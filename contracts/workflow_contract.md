# Workflow Contract System

## Purpose

This document defines the mandatory workflow behavior contracts for the AI-native presentation cognition system.

The objective is NOT:
workflow flexibility.

The objective IS:
stable cognition orchestration behavior.

Traditional AI systems suffer from:

- unstable workflow switching
- prompt-driven personality drift
- inconsistent communication behavior
- random rendering priorities
- fragmented refinement logic

This contract exists to ensure:

workflows behave as:

```text
persistent cognition operating modes
```

NOT:

```text
temporary prompt styles
```

Workflows define:

how the runtime thinks.

---

A workflow is NOT:

a cognition operating mode.

The runtime MUST preserve:

workflow continuity across:

- generation
- rendering
- evaluation
- refinement
- collaboration

The runtime MUST NOT:

randomly switch cognition modes.
