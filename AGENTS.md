It is an:

- AI-native presentation cognition system
- workflow-driven slide intelligence framework
- cognitive communication architecture
- presentation reasoning engine

The purpose is NOT:
- visual ornament
- template stacking
- aesthetic-only slides

The purpose IS:
- audience cognition optimization
- narrative reasoning
- scientific communication
- teaching clarity
- mechanism explanation
- visual semantic mapping
- presentation intelligence

---

# Core Philosophy

A good presentation does NOT show
how much the presenter knows.

It helps the audience understand
as easily as possible.

Presentation design is:
- cognitive engineering
- semantic communication
- guided visual reasoning

All slide generation decisions
must prioritize:

1. understanding
2. clarity
3. cognitive flow
4. narrative coherence
5. visual hierarchy

NOT decoration.

---

# System Architecture

The repository operates as a:

Multi-workflow orchestration system.

AI agents must:

1. identify presentation scenario
2. identify audience
3. identify cognitive objective
4. identify narrative structure
5. activate appropriate workflow(s)
6. generate presentation structure
7. generate slide-level cognition
8. optimize cognitive load
9. optimize visual communication

# Slide Generation Principles

Every slide must contain:

- one cognitive objective
- one focal point
- one key takeaway

Avoid:

- multi-focus slides
- information dumping
- disconnected visuals
- unnecessary decoration

---

# Narrative Structure

Preferred presentation structure:

Problem
→ Importance
→ Principle
→ Method
→ Example
→ Result
→ Summary

Narrative should feel:

- progressive
- natural
- easy to follow
- cognitively smooth

---

# Cognitive Load Rules

Always ask:

Can audience understand this
within 3 seconds?

If NO:
- simplify
- reduce text
- reduce colors
- reduce complexity
- strengthen hierarchy

Reduction is preferred over addition.

---

# Visual Hierarchy Rules

Always establish:

Title
↓
Core visual
↓
Key takeaway
↓
Supporting information

Most important elements should be:

- larger
- clearer
- more central
- visually dominant

Never allow all elements
to compete equally.

---

# Visual Semantic Mapping

Colors must communicate meaning.

Examples:

Green:
- agriculture
- ecology
- biology
- sustainability

Blue:
- GIS
- remote sensing
- technology
- data systems

Orange:
- warning
- stress
- anomaly
- emphasis

Red:
- risk
- damage
- limitation

Visual semantics are functional,
NOT decorative.

---

# Reading Flow Design

Slides must guide eye movement naturally.

Preferred reading flow:

Title
→ visual focus
→ key insight
→ supporting details

Audience attention should never feel lost.

---

# Teaching Adaptation Rules

Teaching-oriented slides should:

- explain visually
- reduce abstraction
- simplify concepts
- emphasize intuitive understanding

Preferred elements:

- diagrams
- process arrows
- comparisons
- annotated visuals
- progressive explanation

Avoid:

- textbook-style paragraphs
- overloaded bullet lists
- formula-heavy layouts

---

# Scientific Mechanism Rules

Scientific mechanism slides should:

- show causal chain clearly
- separate stages visually
- emphasize mechanism progression
- reveal relationships spatially

Preferred structures:

- flow diagrams
- layered mechanisms
- progressive causal systems
- interaction networks

Avoid:

- disconnected charts
- visually fragmented layouts
- unexplained complexity

---

# Interview Optimization Rules

Interview presentations should:

- amplify strengths
- reduce weakness exposure
- appear structured
- appear reliable
- appear professional

If presenter lacks:
- theoretical depth
→ simplify theory
→ emphasize applications

If presenter lacks:
- publication strength
→ emphasize methodology
→ emphasize innovation
→ emphasize future potential

# AI Execution Order

AI agents should execute tasks in this order:

1. scenario analysis
2. audience modeling
3. workflow routing
4. cognitive objective extraction
5. narrative structure generation
6. slide classification
7. layout reasoning
8. visual semantic mapping

# Final Validation Rules

Before final output, verify:

- Is narrative coherent?
- Is cognitive load acceptable?
- Is audience understanding prioritized?
- Is visual hierarchy clear?
- Does each slide have one focal point?
- Is semantic communication effective?
- Is unnecessary complexity removed?

If NOT:
iterate and simplify.

---

- generate decorative-only slides
- overload slides with text
- use meaningless visual effects
- create visually noisy layouts
- prioritize aesthetics over cognition
- use random color systems
- generate disconnected slide sequences
- imitate generic commercial templates
