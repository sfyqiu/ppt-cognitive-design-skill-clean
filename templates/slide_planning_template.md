# Slide Planning Template

## Purpose

This template defines how AI should
plan a presentation slide
before generating visual layouts.

The goal is:

- stable structure
- clear communication
- audience adaptation
- controllable rendering

NOT random slide generation.

---

# Slide Identity

## Slide Type

Examples:

- cover
- intro
- mechanism
- comparison
- process
- workflow
- result
- summary
- thank-you

---

## Slide Goal

Define:

What should audience understand
after viewing this slide?

Only one primary goal is allowed.

---

## Audience

Examples:

- undergraduate students
- graduate students
- academic experts
- interview committee
- mixed audience

Audience determines:

- density
- terminology
- visual complexity
- explanation depth

---

# Cognitive Strategy

## Key Message

Define the single most important idea.

Rule:

One slide
=
one core message.

---

## Cognitive Density

Options:

- low
- medium
- high

Adjust according to:

- audience expertise
- presentation scenario
- explanation complexity

---

## Narrative Role

Examples:

- introduce problem
- explain mechanism
- compare methods
- transition section
- summarize contribution

---

# Visual Planning

## Visual Focus

Define the dominant visual element.

Examples:

- mechanism diagram
- process flow
- comparison chart
- conceptual framework
- GIS map
- annotated figure

---

## Layout Strategy

Examples:

- left text + right figure
- top title + bottom flow
- central mechanism
- comparison columns
- progressive process

---

## Visual Emphasis

Highlight:

- mechanism
- result
- innovation
- comparison
- conclusion

Avoid multiple competing focal points.

---

# Content Constraints

## Must Include

Define critical elements.

Examples:

- key mechanism
- one supporting figure
- conclusion sentence

---

## Avoid

Examples:

- dense paragraphs
- unrelated figures
- decorative clutter
- excessive annotations

---

# Revision Awareness

Slides should support:

- local modification
- figure replacement
- text refinement
- layout adjustment

without breaking:

- narrative consistency
- visual consistency
- audience adaptation

---

# Final Validation

Before generating slide:

Check:

- Is slide purpose clear?
- Is focal point identifiable?
- Is density appropriate?
- Is audience adaptation correct?
- Is layout strategy stable?

If NO:

simplify before rendering.

---

# Template Philosophy

Good slide planning reduces
generation instability.

A presentation slide should be:

intentional,
focused,
and cognitively clear.
