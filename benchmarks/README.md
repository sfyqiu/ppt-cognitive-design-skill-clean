# Presentation Benchmark System

## Purpose

This directory stores benchmark cases
for evaluating AI-generated presentations.

Benchmarks are used to:
- evaluate cognitive quality
- compare generation strategies
- validate workflow effectiveness
- improve skill routing
- optimize presentation reasoning

---

# Benchmark Categories

## teaching/

Evaluate:
- teaching clarity
- cognitive simplicity
- educational effectiveness
- explanation smoothness

---

## academic/

Evaluate:
- scientific mechanism clarity
- research communication
- scientific hierarchy
- logical coherence

---

## interview/

Evaluate:
- strategic communication
- professional positioning
- credibility construction
- narrative control

---

# Evaluation Dimensions

All benchmark cases should evaluate:

- cognitive load
- readability
- visual hierarchy
- semantic consistency
- narrative coherence
- audience adaptation
- mechanism clarity
- strategic communication effectiveness

---

# Benchmark Philosophy

A benchmark should NOT measure:
- decorative richness
- animation complexity
- visual novelty alone

A benchmark SHOULD measure:
- understanding efficiency
- cognitive smoothness
- scientific interpretability
- communication effectiveness

---

# Long-Term Goal

This benchmark system should evolve into:
- AI presentation evaluation framework
- cognitive slide quality system
- presentation reasoning benchmark
- adaptive workflow evaluation platform
