# Iterative Refinement Flow System

## Purpose

This document defines the iterative refinement execution flow used by the AI-native presentation cognition system.

The objective is NOT:
continuous regeneration.

The objective IS:
persistent cognition refinement collaboration.

Traditional AI systems refine presentations through:

```text
feedback
↓
regenerate_slide
↓
replace_output
```

This system refines presentations through:

```text
feedback
↓
cognition_diagnosis
↓
localized_refinement
↓
renderer_coordination
↓
communication_validation
↓
continuity_preservation
↓
persistent_iteration
```

The iterative refinement flow exists to ensure:

refinement improves communication
without destroying presentation identity.
