# Runtime Entry System

## Purpose

This document defines how an external agent enters the AI-native presentation cognition runtime.

The objective is NOT:
immediate presentation generation.

The objective IS:
controlled cognition runtime initialization.

Traditional AI systems behave as:

```text
prompt
↓
generate
```

This runtime behaves as:

```text
intent understanding
↓
workflow dispatch
↓
audience reasoning
↓
cognition initialization
↓
artifact orchestration
↓
rendering execution
↓
persistent collaboration
```

The runtime entry system exists to ensure:

every agent entering the system
obeys cognition runtime laws.

---

# Core Runtime Entry Philosophy

Agents MUST NOT:

immediately generate slides.

Agents MUST:

first establish:

- cognition context
- workflow identity
- audience model
- communication goals
- presentation identity
- rendering strategy

Runtime entry initializes:

persistent cognition state.
