# Presentation Revision Engine

## Purpose

This engine controls how
the AI presentation cognition system
should revise,
refine,
and stabilize
presentations through iterative cognitive alignment.

The objective is NOT:
continuous redesign.

The objective IS:
targeted communication improvement
through controlled visual and cognitive refinement.

Revision should improve:

- interpretation speed
- hierarchy clarity
- cognitive accessibility
- scientific readability
- professional stability

while preserving:

- presentation identity
- narrative continuity
- visual atmosphere
- communication intent

Revision should feel:

intentional,
minimal,
and strategically controlled.

---

# Core Revision Philosophy

Presentation revision is NOT:

random redesign.

It IS:

cognitive correction engineering.

The engine must first determine:

- what communication failed
- what visual behavior caused failure
- what hierarchy became unstable
- what cognitive friction emerged

Then:

perform the smallest revision
with the highest communication impact.

Avoid:

- unnecessary regeneration
- unrelated redesign
- visual personality switching
- narrative destabilization

Revision should preserve:

communication coherence.

# Revision Diagnosis Engine

Before revising,
the system must determine:

- what failed
- why it failed
- where cognitive friction appears
- whether the problem is local or global

Possible diagnosis categories:

- hierarchy failure
- cognitive overload
- weak mechanism interpretation
- poor teaching accessibility
- unstable professionalism
- narrative fragmentation
- visual atmosphere inconsistency
- focal competition

Revision must NEVER begin
before diagnosis.

---

# Local Revision Principle

Default behavior:

local refinement.

The engine should preserve:

- existing slide rhythm
- visual atmosphere
- narrative continuity
- typography systems
- semantic color behavior

Avoid:

- full regeneration
- unrelated restructuring
- unnecessary redesign escalation

Local correction is preferred
unless global redesign is required.

# Hierarchy Revision Engine

When hierarchy fails,
prioritize correcting:

- dominant message visibility
- focal clarity
- evidence subordination
- reading order
- visual competition

Preferred revision actions:

- enlarge dominant elements
- reduce secondary visibility
- increase whitespace isolation
- stabilize visual grouping

Avoid:

- adding more elements
- increasing visual density
- equal-weight redesign

Hierarchy correction should improve:

attention guidance.

---

# Cognitive Density Revision Engine

If overload appears,
prioritize reducing:

- simultaneous concepts
- annotation quantity
- evidence competition
- terminology density
- visual fragmentation

Preferred actions:

- split slides
- enlarge focal visuals
- simplify annotations
- reduce local complexity
- increase breathing room

Avoid:

adding more content.

Reduction is preferred.

---

# Mechanism Revision Engine

When scientific mechanisms fail,
prioritize improving:

- directional flow
- causal clarity
- process hierarchy
- visual guidance
- mechanism readability

Preferred revision actions:

- strengthen arrows
- simplify pathways
- enlarge mechanisms
- reduce competing evidence
- isolate process stages

Avoid:

- tangled workflows
- excessive scientific detail
- decorative scientific graphics

Mechanism revision should improve:

scientific interpretability.

---

# Teaching Revision Engine

When learners struggle,
prioritize improving:

- conceptual accessibility
- visual explanation
- progressive teaching flow
- terminology simplicity
- diagram readability

Preferred revision actions:

- enlarge diagrams
- simplify wording
- reduce abstraction
- isolate one dominant concept
- strengthen annotation guidance

Avoid:

- oversimplification
- removing core meaning
- introducing childish visuals

Teaching revision should improve:

understanding speed.

---

# Professionalism Revision Engine

When presentations feel unprofessional,
prioritize correcting:

- typography instability
- decorative excess
- hierarchy imbalance
- aggressive visuals
- inconsistent spacing

Preferred revision actions:

- reduce visual aggression
- stabilize hierarchy rhythm
- simplify layouts
- improve editorial calmness
- reduce decorative styling

Avoid:

- startup aesthetics
- infographic contamination
- dashboard-style redesign

Professional revision should improve:

trust.

---

# Reading Flow Revision Engine

Revision should preserve
or improve:

stable eye movement rhythm.

Preferred reading behavior:

Title
↓
Dominant visual
↓
Directional explanation
↓
Supporting evidence
↓
Interpretation

Avoid:

- scattered focal regions
- disconnected visual islands
- unstable navigation patterns

Reading should feel:

guided
and cognitively smooth.

---

# Whitespace Revision Engine

Whitespace should be revised when:

- slides feel crowded
- hierarchy becomes unstable
- mechanisms lose readability
- interpretation becomes difficult

Preferred whitespace actions:

- enlarge margins
- isolate dominant visuals
- reduce compression
- separate evidence zones

Whitespace revision improves:

cognitive calmness.

Revision must preserve:

- presentation identity

Avoid:

- visual personality drift

- unrelated redesign behavior

The revised deck should still feel:

# Anti-AI-Slop Revision Rules

Strictly avoid:

- random redesign generation
- startup visual contamination
- infographic-heavy restructuring
- glowing AI visual additions
- decorative complexity escalation

Revision should feel:

human-guided,
editorially stable,
and strategically restrained.

---

# Evaluation Loop

After revision,
the engine must re-evaluate:

- cognitive clarity
- focal hierarchy
- density stability
- scientific readability
- teaching accessibility
- professionalism
- visual calmness

If revision still fails:

continue refinement,
NOT regeneration.

---

# Final Validation

Before finalizing revision:

Check:

- Was the actual communication problem solved?
- Is the dominant message clearer?
- Is cognitive pressure reduced?
- Is hierarchy more stable?
- Is reading flow smoother?
- Is presentation identity preserved?
- Does the deck remain visually coherent?

If NO:

continue iterative refinement.

iterative cognitive alignment
