# Prompt Chaining System

This document defines how the PPT Cognitive Design framework executes chained reasoning.

The system should NOT directly generate slides from raw user prompts.

Instead, it should progressively transform user requirements into cognitively optimized presentation structures.

---

# Core Principle

Presentation intelligence requires:

- staged reasoning
- progressive refinement
- adaptive cognition
- iterative optimization

The system must think before generating.

---

# Global Prompt Chain

```text
User Request
    ↓
Requirement Parsing Prompt
    ↓
Scenario Reasoning Prompt
    ↓
Audience Reasoning Prompt
    ↓
Design Strategy Decision Prompt
    ↓
Narrative Construction Prompt
    ↓
Slide Planning Prompt
    ↓
Layout Recommendation Prompt
    ↓
Visual Semantic Prompt
    ↓
Cognitive Optimization Prompt
    ↓
Presentation Evaluation Prompt
    ↓
Final Presentation Output
```

---

# Stage 1 — Requirement Parsing

Purpose:

- identify hidden goals
- extract constraints
- infer presentation type
- detect risks
- classify content complexity

Outputs:

- presentation scenario
- audience profile
- duration
- objectives
- technical density
- risk factors

---

# Stage 2 — Scenario Reasoning

Purpose:

- identify presentation context
- infer professionalism requirements
- determine narrative expectations
- estimate explanation style

Examples:

- teaching interview
- academic defense
- business presentation
- scientific storytelling

---

# Stage 3 — Audience Reasoning

Purpose:

- estimate expertise level
- estimate cognitive tolerance
- determine terminology complexity
- determine pacing strategy

Audience cognition controls presentation density.

---

# Stage 4 — Design Strategy Decision

Purpose:

- determine presentation style
- determine visual density
- determine teaching intensity
- determine emotional tone
- determine cognitive compression level

Different contexts require different strategies.

---

# Stage 5 — Narrative Construction

Purpose:

- build storyline
- organize information progression
- establish cognitive rhythm
- reduce narrative confusion

Narrative should guide audience understanding progressively.

---

# Stage 6 — Slide Planning

Purpose:

- determine slide sequence
- organize sections
- manage pacing
- allocate information density

Each slide should have:

- one focal objective
- one key takeaway
- one dominant attention region

---

# Stage 7 — Layout Recommendation

Purpose:

- match content to layout structure
- control visual hierarchy
- improve readability
- reduce navigation complexity

Layouts should support cognition.

---

# Stage 8 — Visual Semantic Mapping

Purpose:

- apply semantic color systems
- establish visual consistency
- reinforce meaning through visuals
- strengthen visual hierarchy

Visuals should communicate meaning.

---

# Stage 9 — Cognitive Optimization

Purpose:

- simplify complexity
- reduce overload
- increase readability
- stabilize visual rhythm
- improve audience comfort

Optimization means removing unnecessary complexity.

---

# Stage 10 — Presentation Evaluation

Purpose:

- evaluate cognitive efficiency
- evaluate audience adaptation
- evaluate narrative quality
- evaluate visual hierarchy
- evaluate presentation rhythm

The system should continuously refine presentation quality.

---

# Chaining Philosophy

Each stage should refine the output of the previous stage.

The system should progressively increase presentation intelligence.

---

# Final Principle

Presentation generation is NOT:

single-step slide creation.

It is:

multi-stage cognitive reasoning.
