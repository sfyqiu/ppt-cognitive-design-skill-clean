# Presentation Output Structure System

This system defines how AI-generated presentation results should be structured.

Presentation intelligence requires structured communication.

The AI should NOT output chaotic design suggestions.

The AI should output:

progressively organized presentation reasoning.

---

# Core Principle

Output structure should mirror cognitive structure.

The audience should understand:

- why the presentation is designed this way
- how the structure supports cognition
- how visual strategy supports understanding

---

# Standard Output Workflow

The AI should generate presentation outputs in the following order:

```text
1. Presentation Positioning
2. Audience Analysis
3. Risk Analysis
4. Narrative Strategy
5. Slide Structure
6. Layout Recommendations
7. Visual Semantic Strategy
8. Cognitive Optimization Suggestions
9. Presentation Style Summary
```

---

# 1. Presentation Positioning

Purpose:

Define:

- presentation identity
- professionalism level
- interdisciplinary positioning
- communication objective

The positioning should align with audience expectations.

---

# 2. Audience Analysis

Purpose:

Explain:

- audience expertise level
- cognitive tolerance
- terminology tolerance
- attention characteristics
- emotional expectations

Presentation complexity should adapt accordingly.

---

# 3. Risk Analysis

Purpose:

Identify:

- questioning risks
- cognitive overload risks
- narrative risks
- audience mismatch risks
- visual confusion risks

The system should proactively reduce presentation failure probability.

---

# 4. Narrative Strategy

Purpose:

Explain:

- storytelling structure
- information progression
- pacing rhythm
- explanation sequence
- transition logic

Narrative should guide audience cognition progressively.

---

# 5. Slide Structure

Purpose:

Generate:

- slide-by-slide structure
- section hierarchy
- cognitive sequencing
- pacing distribution

Each slide should contain:

- one focal objective
- one dominant takeaway

---

# 6. Layout Recommendations

Purpose:

Recommend:

- slide layouts
- visual hierarchy
- diagram structures
- comparison systems
- workflow structures

Layouts should support comprehension.

---

# 7. Visual Semantic Strategy

Purpose:

Define:

- color semantics
- visual tone
- hierarchy style
- diagram style
- spacing rhythm

Visual systems should communicate meaning.

---

# 8. Cognitive Optimization Suggestions

Purpose:

Identify:

- simplification opportunities
- overload risks
- visual improvements
- readability improvements
- hierarchy improvements

Optimization should continuously reduce cognitive burden.

---

# 9. Presentation Style Summary

Purpose:

Summarize:

- presentation atmosphere
- professionalism level
- audience experience
- visual rhythm
- communication effectiveness

The summary should describe the final presentation personality.

---

# Output Style Rules

The AI should:

- maintain structured hierarchy
- use concise explanation
- avoid chaotic suggestion dumping
- explain reasoning clearly
- maintain cognitive readability

---

# Final Principle

Presentation output succeeds when:

the user understands both:

- WHAT to design
- and WHY it should be designed that way.
