# Teaching Cognition Engine

## Purpose

This engine controls how
the AI presentation cognition system
should generate
teaching-oriented presentations
through progressive cognitive communication.

The objective is NOT:
maximum information delivery.

The objective IS:
maximum understandable learning.

Teaching presentations should help learners:

- understand rapidly
- follow naturally
- interpret visually
- remember core concepts
- reduce cognitive pressure

Teaching communication should feel:

guided,
intuitive,
and cognitively accessible.

---

# Core Teaching Philosophy

Teaching presentations are NOT:

compressed textbooks.

They are:

learning guidance systems.

The engine must prioritize:

- conceptual clarity
- visual explanation
- progressive cognition
- educational readability
- interpretation accessibility

NOT:

- academic density
- terminology accumulation
- formula-heavy communication
- manuscript-style explanation

Teaching communication should reduce:

interpretation friction.

# Cognitive Density Engine

Teaching presentations require:

low-to-medium density.

One slide should communicate:

- one concept
- one process
- one takeaway

Avoid:

- simultaneous concept overload
- dense terminology
- excessive explanation layers
- multiple competing visuals

If overloaded:

- split concepts
- enlarge diagrams
- simplify annotations
- increase whitespace
- reduce terminology

Educational accessibility
is more important
than information quantity.

---

# Final Validation

Before finalizing:

Check:

- Can learners understand within seconds?
- Is the dominant concept obvious?
- Is the diagram stronger than the text?
- Is cognitive density controlled?
- Is terminology accessible?
- Is reading flow smooth?
- Does the slide feel educationally calm?

If NO:

simplify,
reduce,
and rebalance.
