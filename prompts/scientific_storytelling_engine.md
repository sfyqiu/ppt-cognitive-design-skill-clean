# Scientific Storytelling Engine

## Purpose

This engine controls how
the AI presentation cognition system
should construct,
sequence,
and communicate
scientific narratives.

The objective is NOT:
result accumulation.

The objective IS:
mechanism-centered scientific storytelling
through controlled cognitive interpretation.

Scientific storytelling should help audiences:

- understand the scientific problem
- follow mechanism evolution
- connect evidence logically
- trust scientific reasoning
- remember the scientific contribution

Scientific storytelling should feel:

guided,
inevitable,
and scientifically coherent.

---

# Core Scientific Philosophy

Scientific presentations are NOT:

paper screenshots.

They are:

scientific interpretation systems.

The engine must prioritize:

- mechanism communication
- causal explanation
- evidence hierarchy
- scientific readability
- editorial calmness

NOT:

- publication-density dumping
- isolated chart accumulation
- excessive statistical display
- decorative scientific aesthetics

# Scientific Narrative Structure

Preferred narrative progression:

Scientific Problem
↓
Research Limitation
↓
Scientific Hypothesis
↓
Experimental Strategy
↓
Mechanism Explanation
↓
Evidence Validation
↓
Interpretation
↓
Contribution
↓
Scientific Significance

The narrative should feel:

- progressive
- evidence-supported
- scientifically inevitable
- cognitively smooth

Avoid:

- disconnected evidence
- abrupt mechanism transitions
- fragmented result sections
- conclusion-first storytelling

---

# Scientific Problem Engine

The opening section must establish:

- why the problem matters
- what scientific limitation exists
- why current approaches fail
- what gap remains unresolved

The audience should rapidly understand:

why this research exists.

Preferred communication behavior:

- concise framing
- visual scientific context
- controlled terminology
- focused scientific tension

Avoid:

- literature review dumping
- excessive citations
- abstract problem statements

Scientific openings should feel:

focused
and interpretable.

---

# Hypothesis Engine

- mechanism expectation

- research direction

Preferred hypothesis behavior:

Problem
→ Mechanism Assumption

Avoid:

- vague scientific claims

- unsupported speculation

Hypotheses should feel:

scientifically motivated.

# Mechanism Storytelling Engine

dominant visual hierarchy.

visually interpretable.

# Scientific Density Engine

The engine must maintain:

controlled medium-to-high density.

One slide should communicate:

- one mechanism
- one interpretation
- one scientific transition

Avoid:

- simultaneous multi-mechanism explanation
- evidence overload
- fragmented scientific focus

If overloaded:

- split scientific layers
- enlarge mechanism hierarchy
- reduce secondary evidence
- simplify annotations

Scientific understanding
is more important
than information quantity.

---

2. evidence integration

4. whitespace rhythm
5. annotation restraint
6. scientific pacing

- adding more scientific detail
- decorative redesign
- unnecessary complexity escalation

# Final Validation

Before finalizing:

Check:

- Is the scientific problem obvious?
- Is the mechanism immediately interpretable?
- Does evidence support the narrative naturally?
- Is scientific density controlled?
- Is whitespace sufficient?
- Does the presentation feel editorially mature?
- Does the narrative avoid fragmented science communication?

If NO:

refine again.
