# Presentation Generation Engine

## Purpose

This engine controls how
the AI presentation cognition system
should generate presentations
through coordinated cognitive orchestration.

The objective is NOT:
template-style slide generation.

The objective IS:
high-quality cognitive communication engineering.

This engine must coordinate:

- workflow systems
- skill routing
- style routing
- visual cognition references
- evaluation systems
- grounded examples
- rendering behavior

Presentation generation should behave as:

intentional communication design.

---

# Core System Philosophy

Presentation generation is NOT:

Prompt
→ Slides

Presentation generation IS:

Scenario Analysis
↓
Audience Modeling
↓
Skill Routing
↓
Style Routing
↓
Cognitive Planning
↓
Narrative Planning
↓
Visual Hierarchy Planning
↓
Mechanism Structuring
↓
Rendering Planning
↓
Evaluation
↓
Refinement

The system must prioritize:

communication effectiveness
over visual decoration.

---

# Global Generation Rules

The system must ALWAYS:

- determine presentation scenario
- determine audience structure
- determine cognitive objective
- determine acceptable density
- determine professionalism requirements
- determine visual atmosphere
- determine communication strategy

Generation must NEVER begin
before routing and planning.

# Narrative Planning Engine

The engine must establish:

Problem
↓
Mechanism
↓
Evidence
↓
Interpretation
↓
Contribution

Narratives should feel:

- progressive
- inevitable
- guided
- coherent
- cognitively smooth

Avoid:

- disconnected sections
- evidence dumping
- fragmented storytelling
- abrupt transitions

---

# Hierarchy Planning Engine

Every slide must establish:

- dominant message
- dominant visual
- supporting structure
- controlled annotations

One slide should communicate:

- one mechanism
- one concept
- one contribution

Avoid:

- multi-focus layouts
- equal-weight composition
- dashboard-style density

---

# Visual Atmosphere Engine

The system should prioritize:

- editorial calmness
- restrained professionalism
- cognitive readability
- visual stability

- startup aesthetics
- infographic-heavy composition
- glowing AI visuals
- decorative gradients
- excessive icon systems

Presentations should feel:

human-designed,

and cognitively mature.

# Mechanism Communication Engine

Scientific mechanisms should be:

- visually dominant
- directionally guided
- causally interpretable
- whitespace isolated

Preferred mechanism behavior:

- arrows
- flow hierarchy
- annotation restraint
- simplified causal chains

Avoid:

- tangled workflows
- dense process systems
- screenshot-based mechanisms

Mechanisms should explain:

relationships,
not complexity.

Typography should feel:

- editorial
- stable

- stable hierarchy scaling
- restrained emphasis

Avoid:

- decorative typography
- aggressive headings
- dense paragraphs

Typography supports:

communication clarity.

# Revision Trigger Rules

If evaluation fails,
the engine must revise:

- hierarchy
- spacing
- density
- annotations
- mechanism flow
- narrative smoothness

Avoid:

adding more content.

Reduction is preferred.

---

# Rendering Rules

Generated presentations must remain:

- editable
- hierarchy-safe
- revision-friendly
- logically grouped
- structurally stable

Avoid:

- flattened exports
- screenshot-heavy composition
- unstable grouping systems

Rendering must support:

future refinement.

---

# Final Validation

Before finalizing:

Check:

- Is the communication objective obvious?
- Is the dominant message immediately visible?
- Is cognitive load controlled?
- Is hierarchy stable?
- Is the visual atmosphere coherent?
- Is mechanism interpretation clear?
- Does the presentation avoid AI presentation clichés?

If NO:

refine again.
