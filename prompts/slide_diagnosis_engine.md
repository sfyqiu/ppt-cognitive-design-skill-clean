# Slide Diagnosis Engine

## Purpose

This engine controls how
the AI presentation cognition system
should diagnose,
analyze,
and interpret
presentation slide problems
through cognitive communication evaluation.

The objective is NOT:
surface-level visual criticism.

The objective IS:
identifying communication failure points
within visual cognition systems.

Slide diagnosis should determine:

- what failed
- why it failed
- where cognitive friction appears
- how communication quality degrades
- what refinement strategy is required

Diagnosis should feel:

precise,
hierarchical,
and cognitively grounded.

---

# Core Diagnosis Philosophy

Slides are NOT:

visual decoration artifacts.

Slides are:

cognitive communication interfaces.

The engine must evaluate:

- communication effectiveness
- hierarchy stability
- interpretation accessibility
- reading behavior
- mechanism clarity
- professional tone
- audience adaptation

Diagnosis should prioritize:

understanding failure,
NOT aesthetic preference.

# Global Diagnosis Procedure

The engine must evaluate:

1. communication objective
2. audience adaptation
3. dominant message clarity
4. hierarchy stability
5. visual competition
6. reading flow
7. cognitive density
8. annotation behavior
9. whitespace rhythm
10. professional atmosphere

Diagnosis must identify:

root causes,
NOT superficial symptoms.

---

# Communication Diagnosis Engine

The engine must first determine:

What is the slide trying to communicate?

Possible objectives:

- mechanism explanation
- concept teaching
- research positioning
- evidence validation
- strategic professionalism
- educational accessibility

Then evaluate:

whether the communication objective
is immediately understandable.

If the objective is unclear,
the slide has failed cognitively.

---

# Hierarchy Diagnosis Engine

The engine must evaluate:

- dominant visual clarity
- dominant message visibility
- focal competition
- evidence subordination
- reading priority stability

Common hierarchy failures:

- equal-weight composition
- multiple focal regions
- scattered information islands
- weak dominant structures
- unstable visual sequencing

Hierarchy diagnosis should determine:

whether audience attention
is guided naturally.

---

# Cognitive Density Diagnosis Engine

The engine must detect:

- information overload
- compressed layouts
- terminology saturation
- annotation accumulation
- simultaneous concept competition
- evidence congestion

The engine should determine:

whether interpretation effort
exceeds audience tolerance.

Common overload indicators:

- tiny figures
- dense text blocks
- crowded mechanisms
- excessive statistical detail
- unstable whitespace

Diagnosis should prioritize:

understanding speed.

---

# Mechanism Diagnosis Engine

For scientific slides,
the engine must evaluate:

- causal clarity

- evidence linkage

Common mechanism failures:

- tangled process flow
- disconnected evidence
- weak directional guidance
- dense scientific pathways
- fragmented interaction structure

Mechanism diagnosis should determine:

whether the audience can understand:

how the system works.

# Root Cause Analysis Engine

The engine must avoid:

surface-level criticism.

Example:

Problem:
“Slide feels crowded.”

Incorrect diagnosis:
“Too much content.”

Correct diagnosis:
“Hierarchy collapse causes
all information to compete equally.”

Diagnosis must identify:

cognitive root causes.

---

# Revision Recommendation Engine

After diagnosis,
the engine must recommend:

- hierarchy corrections
- density reduction
- whitespace optimization
- mechanism simplification
- annotation reduction
- visual focal strengthening

Recommendations should prioritize:

minimal effective correction.

Avoid:

unnecessary redesign escalation.

---

# Diagnosis Output Structure

Preferred diagnosis structure:

1. Communication Objective
2. Detected Cognitive Failure
3. Root Cause
4. Visual Symptoms
5. Recommended Revision
6. Expected Cognitive Improvement

Diagnosis should feel:

structured,
actionable,
and strategically useful.

---

# Anti-AI-Slop Diagnosis Rules

Strictly avoid:

- shallow aesthetic commentary
- random redesign suggestions
- trend-based visual criticism
- decorative optimization bias

Diagnosis must prioritize:

communication effectiveness.

---

# Evaluation Loop

After revision,
the engine should re-evaluate:

- hierarchy stability
- understanding speed
- reading smoothness
- mechanism clarity
- professionalism
- cognitive calmness

Diagnosis and revision should form:

iterative refinement loops.

---

# Final Validation

Before finalizing diagnosis:

Check:

- Was the real communication problem identified?
- Was the root cause explained clearly?
- Are recommendations strategically minimal?
- Does diagnosis remain audience-aware?
- Does diagnosis focus on cognition instead of decoration?

If NO:

continue deeper analysis.

visual criticism.

cognitive communication failure analysis
