# Command — paper_to_ppt

## Purpose

This command converts
scientific papers,
research manuscripts,
or scientific project content
into cognitively optimized presentations.

The objective is NOT:
paper compression.

The objective IS:
mechanism-centered scientific communication
through structured cognitive orchestration.

The generated presentation should help audiences:

- understand the scientific problem
- follow mechanism logic
- interpret evidence efficiently
- recognize scientific contribution
- remember key mechanisms

Scientific presentations should feel:

editorial,
scientifically grounded,
and cognitively interpretable.

---

# Command Activation

Use this command when the user requests:

- paper to PPT conversion
- scientific presentation generation
- conference slides
- thesis defense slides
- manuscript presentation

- research seminar slides

Typical requests include:

- “Convert my paper into slides”
- “Generate a scientific PPT”
- “Create conference presentation”
- “Build thesis defense slides”
- “Explain this mechanism visually”

# Slide Planning Rules

Each slide should communicate:

- one mechanism
- one interpretation
- one scientific transition

Preferred slide types:

- problem framing slides
- mechanism explanation slides
- evidence interpretation slides
- framework slides
- contribution slides
- application implication slides

Avoid:

- multi-focus slides
- dense chart collections
- equal-weight evidence grids

---

# Mechanism Communication Rules

Mechanisms must dominate:

scientific storytelling.

Preferred mechanism behavior:

- large central diagrams
- directional arrows
- causal hierarchy
- annotation restraint
- whitespace isolation

Avoid:

- tangled scientific workflows
- tiny embedded mechanisms
- dense multi-panel structures

Mechanisms should explain:

relationships,
not complexity.

---

# Evidence Communication Rules

Evidence should support:

mechanism trust.

Preferred evidence behavior:

- selective validation
- concise interpretation
- visually subordinate positioning
- grouped evidence hierarchy

Avoid:

- chart dumping
- dense statistical overload
- evidence-first storytelling

Evidence should answer:

why the mechanism is believable.

---

# Cognitive Density Rules

Scientific density should remain:

controlled medium-to-high.

The command must continuously evaluate:

- readability
- interpretation effort
- mechanism clarity
- audience tolerance

If overloaded:

- split slides
- reduce secondary evidence
- enlarge dominant mechanisms
- simplify annotations
- increase whitespace

Scientific understanding
is more important
than information completeness.

---

- glowing AI science graphics
- dashboard composition

editorial,

# Final Validation

Before finalizing:

Check:

- Is the scientific problem immediately obvious?
- Is the mechanism visually interpretable?
- Is evidence subordinate to the mechanism?
- Is cognitive density controlled?
- Is whitespace sufficient?
- Does the presentation feel editorially mature?
- Does the deck avoid AI scientific clichés?

If NO:

refine again.
