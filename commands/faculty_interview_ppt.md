# Command — faculty_interview_ppt

## Purpose

This command generates
faculty interview presentations
through strategic academic communication orchestration.

The objective is NOT:
self-promotional slide generation.

The objective IS:
building professional trust,
academic credibility,
and future potential perception
through cognitively controlled presentation design.

The generated presentation should help committees feel:

- this candidate can teach
- this candidate can communicate
- this candidate has research potential
- this candidate fits the institution
- this candidate has long-term academic value

Faculty presentations should feel:

editorial,
professionally calm,
and institutionally mature.

---

# Command Activation

Use this command when the user requests:

- faculty interview PPT
- academic recruitment presentation
- job talk slides
- candidate introduction presentation
- teaching + research interview deck
- institutional academic presentation

Typical requests include:

- “Generate my faculty interview PPT”
- “Create academic job talk slides”
- “Build interview presentation”
- “Design faculty recruitment deck”
- “Create teaching demonstration + research presentation”

# Cognitive Density Rules

Faculty interview presentations require:

controlled moderate density.

Each slide should communicate:

- one strategic message
- one professional signal
- one interpretable framework

Avoid:

- publication accumulation
- excessive scientific layering
- multi-focus communication

If overloaded:

- reduce evidence quantity
- enlarge whitespace
- simplify framework hierarchy
- isolate strategic messages

Professional communication quality
is more important
than information quantity.
