# Command — teaching_demo_ppt

## Purpose

This command generates
teaching demonstration presentations
through progressive cognitive communication orchestration.

The objective is NOT:
knowledge compression.

The objective IS:
maximum understandable learning
through visually guided educational communication.

The generated presentation should help learners:

- understand rapidly
- follow naturally
- interpret visually
- remember core concepts
- reduce cognitive pressure

Teaching presentations should feel:

approachable,
visually guided,
and cognitively calm.

---

Use this command when the user requests:

- teaching demonstration PPT
- classroom lecture slides

- educational concept explanation
- teaching-oriented academic slides
- lecture visualization

Typical requests include:

- “Generate teaching demo slides”
- “Create undergraduate lecture PPT”
- “Build teaching presentation”
- “Explain this concept for students”
- “Create visually intuitive teaching slides”

# Concept Communication Rules

Teaching slides should prioritize:

- one dominant concept
- one explanatory process
- one memorable takeaway

Preferred concept behavior:

- simplified wording
- intuitive explanation
- visual-first communication
- conceptual clarity

Avoid:

- multi-concept competition
- dense scientific explanation
- abstract terminology overload

Concept communication should maximize:

understanding speed.

---

# Diagram Communication Rules

Diagrams should dominate:

learning interpretation.

Preferred diagram behavior:

- large explanatory visuals
- simplified process structures
- directional arrows
- semantic color guidance
- isolated conceptual flow

Preferred diagram types:

- conceptual illustrations
- process diagrams
- environmental interaction visuals
- comparison graphics
- simplified mechanism visuals

Avoid:

- publication-style figures
- infographic-heavy systems
- visually noisy educational graphics
- dense scientific pathways

Diagrams should explain:

meaning,
not complexity.

---

# Teaching Accessibility Rules

The command must continuously evaluate:

- terminology difficulty
- abstraction level
- interpretation effort
- learner attention burden

If students may struggle:

simplify aggressively.

Preferred accessibility behavior:

- progressive explanation
- intuitive visuals
- concise teaching text
- guided interpretation

Avoid:

- intimidating scientific density
- excessive formulas
- manuscript-style teaching

Teaching accessibility
takes priority.

---

# Cognitive Density Rules

Teaching presentations require:

low-to-medium density.

Each slide should communicate:

- one concept
- one process
- one takeaway

Avoid:

- dense text blocks
- multiple simultaneous mechanisms
- crowded layouts
- annotation overload

If overloaded:

- split concepts
- enlarge diagrams
- reduce terminology
- increase whitespace
- simplify annotations

Educational readability
is more important
than information quantity.
