# Presentation Design Constraints System

This system defines cognitive and visual constraints for presentation intelligence.

Presentation quality depends not only on:

what should be included

but also on:

what should be avoided.

Constraint systems prevent presentation failure.

---

# Core Principle

Good presentations are usually constrained presentations.

Constraint improves:

- clarity
- readability
- hierarchy stability
- cognitive comfort
- communication efficiency

---

# Cognitive Constraints

The system should avoid:

- excessive simultaneous concepts
- overloaded slides
- dense paragraphs
- excessive terminology
- unstable narrative pacing

Presentation density should remain cognitively manageable.

---

# Visual Constraints

The system should avoid:

- excessive decoration
- visual competition
- inconsistent color systems
- unstable alignment
- crowded layouts
- unnecessary visual effects

Visual complexity should remain controlled.

---

# Narrative Constraints

The system should avoid:

- disconnected storytelling
- abrupt transitions
- information dumping
- repetitive explanation
- unstable pacing

Narrative progression should feel natural.

---

# Teaching Constraints

Teaching-oriented presentations should avoid:

- formula overload
- abstract-first explanation
- excessive academic language
- dense analytical structures
- intimidating visual hierarchy

Teaching should remain approachable.

---

# Academic Constraints

Academic presentations should avoid:

- decorative storytelling
- unsupported claims
- excessive animation
- weak evidence hierarchy
- visual exaggeration

Scientific credibility should remain stable.

---

# Business Constraints

Business presentations should avoid:

- excessive methodology detail
- slow messaging
- crowded dashboards
- technical abstraction overload
- weak value communication

Decision efficiency should remain high.

---

# Scientific Storytelling Constraints

Scientific storytelling presentations should avoid:

- fragmented mechanisms
- isolated charts
- disconnected visual systems
- raw data stacking
- weak conceptual hierarchy

Scientific narratives should remain understandable.

---

# Audience Constraints

Low-expertise audiences require stronger constraints on:

- terminology density
- abstraction level
- information pacing
- slide complexity

High-expertise audiences require constraints on:

- oversimplification
- decorative reductionism
- insufficient rigor

---

# Visual Rhythm Constraints

The system should avoid:

- continuous dense slides
- repetitive layouts
- visual monotony
- uninterrupted analytical structures

Visual rhythm should alternate naturally.

---

# Risk-control Constraints

The system should avoid:

- over-claiming expertise
- unstable interdisciplinary positioning
- excessive technical exposure
- weak explanation transparency

Risk-aware simplification improves presentation resilience.

---

# Constraint Failure Signals

Constraints have failed if:

- the audience feels overwhelmed
- attention becomes unstable
- comprehension declines
- visual fatigue increases
- questioning risk rises

---

# Final Principle

Presentation intelligence succeeds when:

complexity remains controlled
while meaning remains clear.
