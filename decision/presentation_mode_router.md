# Presentation Mode Router

This system determines which presentation intelligence mode should be activated based on user requirements.

Different presentation scenarios require different reasoning systems.

The router dynamically selects the optimal presentation agent.

---

# Core Principle

There is NO universal presentation strategy.

Presentation intelligence must adapt to:

- audience
- scenario
- cognitive requirements
- communication objectives
- presentation risks

---

# Routing Objective

The router should determine:

- which presentation mode to activate
- which reasoning depth to apply
- which narrative strategy to prioritize
- which visual system to use
- which cognitive optimization strategy to emphasize

---

# Available Presentation Modes

## 1. Teaching Presentation Mode

Activate when:

- undergraduate teaching
- classroom presentation
- educational explanation
- teaching interview
- concept demonstration

Prioritize:

- cognitive clarity
- progressive explanation
- diagram-driven layouts
- low cognitive load
- educational pacing

---

## 2. Academic Defense Mode

Activate when:

- thesis defense
- scientific conference
- academic reporting
- research presentation
- methodology explanation

Prioritize:

- logical rigor
- scientific hierarchy
- evidence visibility
- restrained professionalism
- methodological clarity

---

## 3. Business Pitch Mode

Activate when:

- investor pitch
- strategic proposal
- product storytelling
- commercial presentation
- executive communication

Prioritize:

- persuasion
- concise messaging
- emotional pacing
- value communication
- rapid comprehension

---

## 4. Scientific Storytelling Mode

Activate when:

- interdisciplinary research
- mechanism explanation
- complex system communication
- graphical scientific storytelling
- conceptual scientific presentation

Prioritize:

- mechanism visualization
- conceptual clarity
- interdisciplinary accessibility
- visual scientific rhythm
- cognitive simplification

---

# Hybrid Routing Rules

Some presentations require multiple modes.

Example:

Teaching Interview + Scientific Research

The router should combine:

- teaching clarity
- scientific professionalism
- controlled technical depth
- interdisciplinary positioning

Hybrid routing should balance cognitive priorities.

---

# Audience-based Routing

## Low-expertise Audience

Increase:

- teaching-oriented reasoning
- visual explanation
- simplification intensity

Reduce:

- analytical density
- technical abstraction

---

## High-expertise Audience

Increase:

- analytical rigor
- scientific detail
- compressed explanation

Reduce:

- excessive simplification
- decorative storytelling

---

# Risk-aware Routing

If the system detects:

- high questioning risk
- interdisciplinary mismatch
- audience uncertainty
- excessive complexity

Then increase:

- cognitive simplification
- hierarchy clarity
- explanation transparency
- restrained positioning

---

# Routing Failure Signals

Routing has failed if:

- presentation style mismatches audience expectations
- cognitive density becomes inappropriate
- narrative pacing feels unstable
- audience engagement declines
- presentation risks increase

---

# Final Principle

Presentation intelligence succeeds when:

the system activates
the right reasoning mode
for the right audience
under the right scenario.
