# Visual Style System

This system defines how presentation visual styles should adapt to different presentation scenarios.

Visual style is not decoration.

Visual style is cognitive communication.

---

# Core Objective

The visual system should:

- reinforce meaning
- guide attention
- reduce cognitive burden
- establish emotional tone
- improve presentation readability

Visual design should support cognition.

---

# Core Visual Principle

Different presentation scenarios require different visual systems.

There is NO universal visual style.

---

# Teaching-oriented Visual Style

Recommended characteristics:

- bright layouts
- strong whitespace
- simplified hierarchy
- educational friendliness
- intuitive diagrams

Recommended colors:

- blue
- green
- soft neutral tones

Avoid:

- overly dark themes
- excessive visual density
- aggressive contrast
- decorative overload

---

# Academic-defense Visual Style

Recommended characteristics:

- restrained professionalism
- figure-centered hierarchy
- stable alignment
- scientific clarity
- minimal decoration

Recommended colors:

- dark blue
- gray
- muted scientific tones

Avoid:

- excessive animation
- decorative gradients
- visually noisy backgrounds

---

# Business-presentation Visual Style

Recommended characteristics:

- bold hierarchy
- concise messaging
- emotional rhythm
- high-impact visuals
- executive readability

Recommended colors:

- deep blue
- black
- orange emphasis
- high-contrast systems

Avoid:

- academic clutter
- dense information blocks
- low-emphasis layouts

---

# Scientific-storytelling Visual Style

Recommended characteristics:

- mechanism-centered visuals
- layered systems
- semantic arrows
- interdisciplinary balance
- visual progression

Recommended colors:

- blue → systems/technology
- green → ecology/biology
- orange → interactions
- gray → secondary scientific detail

Avoid:

- isolated visual elements
- inconsistent scientific symbols
- fragmented diagram systems

---

# Visual Hierarchy Rules

Every slide should contain:

- one dominant visual region
- one primary attention anchor
- one cognitive focal point

Avoid:

- equal-weight layouts
- scattered attention
- visual competition

---

# Whitespace Strategy

Whitespace should be treated as:

cognitive breathing space.

Whitespace improves:

- readability
- elegance
- comprehension
- attention control

---

# Typography Rules

Typography should prioritize:

- readability
- hierarchy clarity
- visual rhythm
- stable scanning

Avoid:

- excessive font variation
- decorative typography
- dense text blocks

---

# Color Semantics

Recommended semantic mapping:

- blue → technology / trust / professionalism
- green → ecology / agriculture / growth
- orange → emphasis / interaction / warning
- gray → secondary information
- white → cognitive breathing space

Colors should reinforce meaning.

---

# Visual Rhythm Rules

Presentations should alternate between:

- dense information slides
- visual breathing slides
- diagrams
- summaries
- focus slides

Avoid visual monotony.

---

# Final Principle

Visual design succeeds when:

the audience understands naturally
without visual fatigue.
