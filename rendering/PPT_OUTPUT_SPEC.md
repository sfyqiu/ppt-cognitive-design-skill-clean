# PPT Output Specification

## Purpose

This specification defines the expected output structure
for AI-generated PowerPoint presentations.

The objective is to ensure:
- editable slides
- cognitive hierarchy
- semantic consistency
- rendering clarity
- presentation usability

---

# Global Output Structure

AI-generated presentations should contain:

1. reasoning summary
2. selected skills
3. slide-by-slide structure
4. visual hierarchy plan
5. semantic color strategy
6. editable rendering instructions

---

# Required Generation Sequence

AI systems should output:

## Step 1 — Cognitive Analysis

Identify:
- scenario
- audience
- cognitive objective
- slide category
- communication strategy

---

## Step 2 — Skill Routing

Specify:
- primary skill
- secondary skill
- reasoning for orchestration

---

## Step 3 — Slide Planning

For each slide:
- slide purpose
- slide type
- narrative role
- focal message
- cognitive emphasis

---

## Step 4 — Layout Structure

For each slide define:
- title region
- visual region
- annotation region
- takeaway region

Specify:
- reading flow
- dominant visual
- information density

---

# Editable PPT Rules

Slides should be:
- fully editable
- layer-separated
- logically grouped

Text:
- editable text objects

Diagrams:
- editable vector-style shapes

Arrows:
- independent editable connectors

Avoid:
- flattened infographic exports
- screenshot-style rendering
- image-only layouts

---

# Mechanism Slide Specification

Mechanism slides should define:

- mechanism chain
- causal direction
- scientific hierarchy
- annotation structure
- semantic color usage

Preferred flow:
left → right
or
top → bottom

Avoid:
- chaotic visual crossing
- disconnected elements

---

# Teaching Slide Specification

Teaching slides should define:
- simplified explanation flow
- progressive disclosure
- intuitive visual mapping
- low cognitive density

Audience should understand:
within several seconds.

---

# Interview Slide Specification

Interview slides should define:
- strategic message
- credibility focus
- professional hierarchy
- restrained visual style

Avoid:
- excessive self-promotion
- visually aggressive layouts

---

# Scientific Slide Specification

Scientific slides should define:
- mechanism clarity
- evidence hierarchy
- annotation logic
- interpretation guidance

Scientific visuals should support:
- explanation
- interpretation
- communication

NOT:
visual decoration.

---

# Semantic Color Specification

Color usage should be intentional.

Examples:

Green:
- ecology
- agriculture
- biology

Blue:
- scientific systems
- modeling
- remote sensing

Orange:
- stress
- warning
- inhibition

Gray/white:
- professionalism
- restraint
- readability

---

# Final Validation Output

Before final output:
AI should validate:
- cognitive clarity
- visual hierarchy
- narrative coherence
- semantic consistency
- editability

Final question:

Does the presentation support
rapid audience understanding?

If NO:
revise and simplify.

---

# Final Philosophy

A PPT is NOT:
a decorated document.

A PPT IS:
a cognitive communication system.
