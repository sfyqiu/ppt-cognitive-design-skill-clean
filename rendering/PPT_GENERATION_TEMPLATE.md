# PPT Generation Template

## Purpose

This template defines the standard structure
for AI-generated editable PowerPoint slides.

The objective is:
- cognitively optimized slides
- editable presentation structure
- semantic visual communication
- reusable rendering logic

---

# Standard Generation Workflow

AI systems should generate slides in this order:

1. Cognitive Analysis
2. Skill Routing
3. Narrative Planning
4. Slide Hierarchy
5. Visual Semantic Planning
6. Editable PPT Structure
7. Rendering Validation

---

# Slide Generation Format

For each slide:

---

## Slide Title

Define:
- concise title
- dominant scientific or teaching message

---

## Slide Objective

Define:
- what audience should understand
- what audience should remember

Each slide should contain:
- one focal message
- one dominant interpretation

---

## Slide Type

Examples:
- mechanism slide
- process slide
- comparison slide
- summary slide
- methodology slide
- interview positioning slide

---

## Narrative Role

Examples:
- introduce problem
- explain mechanism
- compare scenarios
- summarize conclusion
- establish credibility

---

## Layout Structure

Define:
- title region
- core visual region
- annotation region
- takeaway region

Specify:
- reading flow
- visual hierarchy
- dominant visual focus

---

## Mechanism Structure

If scientific:

define:
- mechanism chain
- directional flow
- layer hierarchy
- causal structure

Prefer:
left-to-right progression.

---

## Visual Semantic Strategy

Specify:
- primary colors
- semantic meaning
- emphasis color usage

Examples:

green:
- biology
- agriculture
- ecology

blue:
- scientific systems
- modeling
- technology

orange:
- stress
- inhibition
- warning

---

## Cognitive Load Control

Specify:
- text density
- visual density
- annotation strategy

Audience should:
understand core meaning quickly.

Reduction is preferred over overload.

---

## Editable PPT Instructions

All elements should remain editable:

- titles
- diagrams
- arrows
- annotations
- icons
- shapes

Prefer:
- vector-style objects
- grouped mechanisms
- separated object layers

Avoid:
- flattened images
- screenshot layouts
- locked infographic exports

---

## Speaker Guidance

Optionally define:
- presentation emphasis
- verbal explanation focus
- strategic communication note

---

# Final Validation

Before final output:

check:
- readability
- hierarchy
- narrative smoothness
- semantic consistency
- editability

Question:

Can audience understand
the dominant message rapidly?

If NO:
simplify layout and structure.

---

# Final Philosophy

PPT generation is NOT:
template decoration.

PPT generation IS:
cognitive communication rendering.
