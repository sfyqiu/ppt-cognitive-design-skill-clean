# Layout Engine Specification

This document defines the adaptive layout intelligence system
for the PPT Cognitive Design Skill framework.

The layout engine is responsible for transforming:

- narrative structure
- semantic hierarchy
- cognitive priorities

into:

- stable visual layouts
- adaptive positioning
- readable presentation structures

---

# Core Principle

Layout is NOT decoration.

Layout is:

- cognitive navigation
- visual attention control
- information hierarchy management
- communication pacing

The layout engine should optimize:

audience understanding efficiency.

---

# Layout Engine Objectives

The engine must:

- avoid overlap
- preserve hierarchy
- maintain visual balance
- control cognitive density
- support progressive reading flow

---

# Rendering Flow

```text
Slide Structure
    ↓
Layout Type Selection
    ↓
Component Priority Analysis
    ↓
Spatial Distribution
    ↓
Collision Detection
    ↓
Alignment Optimization
    ↓
Responsive Rendering
```

---

# Component Priority System

Every slide component must have:

| Priority | Meaning |
|---|---|
| P1 | Primary focus |
| P2 | Secondary support |
| P3 | Supplemental information |
| P4 | Decorative or auxiliary |

---

# Layout Hierarchy Rules

## P1 Components

Examples:

- core mechanism diagram
- key workflow
- central concept

Rules:

- largest visual area
- strongest contrast
- central attention position

---

## P2 Components

Examples:

- supporting explanation
- secondary diagrams
- labels

Rules:

- support P1
- avoid competing emphasis

---

## P3 Components

Examples:

- notes
- citations
- side explanations

Rules:

- visually lighter
- reduced contrast
- peripheral placement

---

# Collision Avoidance System

The layout engine must detect:

- overlapping boxes
- text collisions
- edge overflow
- diagram interference

---

# Collision Handling Strategy

When overlap occurs:

## Priority Rule

Lower-priority components move first.

```text
P3 moves before P2
P2 moves before P1
```

---

## Preferred Adjustment Order

```text
1. Increase spacing
2. Shift auxiliary components
3. Compress secondary regions
4. Resize low-priority elements
5. Re-route layout flow
```

---

# Flow Layout System

## Horizontal Mechanism Flow

Used for:

- causal chains
- scientific processes
- educational explanation

---

# Horizontal Flow Rules

## Minimum Box Spacing

```text
40–80 px
```

---

## Safe Diagram Width

Mechanism flow should not exceed:

```text
75% of slide width
```

Remaining space reserved for:

- notes
- GIS blocks
- explanation panels

---

# Side Panel Rules

Side panels include:

- GIS notes
- teaching notes
- scientific explanation

---

## Side Panel Constraints

Maximum width:

```text
20% of slide width
```

Preferred location:

```text
top-right
or
bottom-right
```

---

# Responsive Layout System

The engine should adapt to:

- long text
- diagram expansion
- varying component counts
- audience complexity

---

# Responsive Adjustment Logic

If diagram becomes too large:

```text
reduce side panel size
```

If side notes overflow:

```text
move notes downward
```

If cognitive density becomes high:

```text
increase whitespace
reduce labels
simplify flow
```

---

# Visual Balance Rules

The slide should maintain:

- left-right balance
- top-bottom stability
- visual breathing space

Avoid:

- crowded corners
- asymmetric overload
- edge collisions

---

# Cognitive Density Constraints

Each slide should contain:

ONE:

- mechanism
- workflow
- comparison
- narrative transition

Avoid:

- multiple simultaneous stories
- overloaded diagrams
- competing visual paths

---

# Reading Path Optimization

Preferred reading path:

```text
top-left
    ↓
center focus
    ↓
support explanation
    ↓
bottom conclusion
```

The layout engine should guide:

audience eye movement naturally.
