# PPT Rendering Protocol

## Purpose

This protocol defines how AI systems
convert cognitive presentation planning
into editable PowerPoint slides.

The objective is NOT:
visual decoration.

The objective IS:
- cognitive readability
- editable structure
- semantic visual communication
- presentation clarity
- rendering consistency

---

# Global Rendering Principles

Slides should:
- remain editable
- preserve hierarchy
- maintain visual clarity
- support verbal presentation
- reduce audience cognitive load

Avoid:
- flattened infographic images
- decorative visual overload
- visually noisy composition
- template-style business aesthetics

---

# Slide Canvas Rules

Default format:
- 16:9 widescreen

Recommended safe margins:
- top: 5%
- bottom: 5%
- left: 6%
- right: 6%

Slides should preserve:
- whitespace
- reading flow
- visual breathing space

Avoid:
- edge-overloaded layouts
- dense object clustering

---

# Visual Hierarchy Rules

Preferred hierarchy:

Title
↓
Core mechanism visual
↓
Key takeaway
↓
Supporting annotations

Hierarchy should be obvious
within several seconds.

Most important elements should be:
- larger
- clearer
- visually dominant

---

# Typography Rules

Recommended hierarchy:

Title:
- large
- bold
- concise

Subtitle:
- smaller
- explanatory
- restrained

Annotations:
- short
- readable
- non-paragraph style

Avoid:
- long text blocks
- dense bullet stacking
- excessive font variation

---

# Mechanism Diagram Rules

Mechanism diagrams should:
- use directional flow
- emphasize causality
- preserve spacing
- avoid visual crossing

Preferred:
- left-to-right flow
- top-to-bottom hierarchy
- layered mechanism organization

Arrows should communicate:
- direction
- influence
- progression

Avoid:
- decorative arrows
- excessive connectors
- chaotic node relationships

---

# Scientific Rendering Rules

Scientific slides should:
- preserve mechanism readability
- separate scientific layers clearly
- emphasize dominant interpretation

Prefer:
- annotation-supported figures
- progressive mechanism layouts
- visualized scientific reasoning

Avoid:
- fragmented scientific charts
- excessive table density
- decorative academic graphics

---

# Teaching Rendering Rules

Teaching slides should:
- simplify abstraction
- strengthen visual explanation
- reduce cognitive friction

Prefer:
- diagrams
- process flows
- comparison layouts
- progressive disclosure

Avoid:
- textbook-style slide rendering
- excessive equations
- overloaded information density

---

# Interview Rendering Rules

Interview slides should:
- appear professional
- appear restrained
- appear coherent
- communicate confidence through structure

Prefer:
- moderate density
- clean whitespace
- strategic visual hierarchy
- concise scientific support

Avoid:
- aggressive layouts
- startup pitch aesthetics
- excessive infographic decoration

---

# Semantic Color Rules

Colors must communicate meaning.

Green:
- agriculture
- ecology
- biological systems

Blue:
- scientific systems
- digital technology
- remote sensing
- modeling

Orange:
- stress
- warning
- inhibition

Gray/white:
- restraint
- professionalism
- cognitive comfort

Avoid:
- random palettes
- excessive saturation
- decorative gradients

---

# Editable PPT Rules

All generated slides should:
- remain fully editable
- preserve object separation
- maintain grouping logic
- support future modification

Avoid:
- flattened visual exports
- image-only slides
- locked layouts

---

# Rendering Validation

Before finalizing:
check:
- readability
- hierarchy clarity
- spacing quality
- cognitive load
- semantic consistency
- editability

Question:

Can the audience quickly understand
the dominant message?

If NO:
simplify rendering.

---

# Final Philosophy

Rendering is NOT decoration.

Rendering is:
visual cognition implementation.
