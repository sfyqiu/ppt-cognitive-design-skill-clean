# Cognitive Load Engine

Presentation design must minimize unnecessary cognitive burden.

A slide should help the audience understand information rapidly and effortlessly.

---

# Core Principle

The audience should understand:

- the main message
- the visual focus
- the narrative direction

within seconds.

If the audience must "work hard" to understand the slide:

the design has failed.

---

# Sources of Cognitive Overload

## Text Overload

Too much text causes:

- reading fatigue
- attention fragmentation
- reduced retention
- audience disengagement

### Symptoms

- full paragraphs
- excessive bullet points
- dense explanations
- small font sizes

### Optimization Strategy

Reduce:

- unnecessary sentences
- repeated explanations
- academic redundancy

Prefer:

- keywords
- short phrases
- visual explanation

---

# Visual Competition

Too many competing elements reduce visual clarity.

### Symptoms

- too many colors
- multiple highlighted regions
- crowded diagrams
- excessive icons
- visual clutter

### Optimization Strategy

Use:

- one primary visual focus
- controlled color hierarchy
- consistent visual rhythm
- intentional whitespace

---

# Information Density Overload

High-density slides increase cognitive stress.

### Symptoms

- too many charts
- excessive data tables
- multi-topic slides
- compressed layouts

### Optimization Strategy

Split content into:

- multiple slides
- progressive explanations
- staged information delivery

---

# Layout Complexity

Complicated layouts increase navigation difficulty.

### Symptoms

- irregular alignment
- inconsistent spacing
- unclear hierarchy
- scattered attention paths

### Optimization Strategy

Use:

- simple grid systems
- stable visual hierarchy
- predictable reading flow
- consistent alignment

---

# Cognitive Breathing Space

Whitespace is not empty.

Whitespace helps:

- visual recovery
- attention reset
- hierarchy separation
- reading comfort

Slides should contain intentional empty regions.

---

# Cognitive Compression Principle

High-quality presentations simplify complexity.

AI should continuously ask:

- Can this slide contain fewer words?
- Can this visual be simpler?
- Can this explanation unfold progressively?
- Can attention focus faster?
- Can audience understanding improve?

---

# Attention Guidance

Slides should guide audience attention intentionally.

Recommended hierarchy:

1. title
2. main visual
3. supporting information
4. details

Avoid:

- equal emphasis everywhere
- multiple simultaneous focal points

---

# Progressive Disclosure

Information should appear progressively.

Do NOT reveal everything simultaneously.

Recommended sequence:

1. context
2. problem
3. explanation
4. evidence
5. takeaway

---

# Cognitive Load Adaptation

Different audiences tolerate different cognitive density.

Examples:

- undergraduate students → low density
- experts → medium/high density
- executives → extremely compressed information

AI should adapt slide complexity accordingly.

---

# Final Principle

Presentation intelligence is not:

# adding more information

It is:

# minimizing audience cognitive effort
