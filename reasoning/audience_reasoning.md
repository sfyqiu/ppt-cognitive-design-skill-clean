# Audience Reasoning Engine

Presentation design must adapt to audience cognition.

Different audiences require different:

- information density
- narrative pacing
- visual complexity
- explanation depth
- semantic emphasis

---

# Undergraduate Students

## Characteristics

- short attention span
- limited domain knowledge
- strong visual dependency
- faster image understanding than text understanding

## Design Strategy

Use:

- large diagrams
- intuitive visuals
- progressive explanations
- minimal text
- simplified terminology

Reduce:

- formula density
- long paragraphs
- theoretical abstraction
- excessive detail

## Recommended Layout

- left text + right image
- step-by-step flow
- centered visual focus

---

# Experts / Professors

## Characteristics

- rapid credibility judgment
- logic-sensitive
- detail-aware
- high information tolerance

## Design Strategy

Use:

- rigorous structure
- clean data visualization
- logical hierarchy
- evidence-supported claims

Reduce:

- decorative graphics
- visual noise
- unnecessary animation

## Recommended Layout

- modular structure
- data-focused layouts
- comparison tables
- structured diagrams

---

# Leadership / Decision Makers

## Characteristics

- extremely limited attention time
- result-oriented
- low tolerance for complexity

## Design Strategy

Use:

- conclusion-first storytelling
- large key messages
- visual hierarchy
- simplified metrics

Reduce:

- detailed methodology
- academic explanation
- dense text blocks

## Recommended Layout

- big-number layouts
- key-point dashboards
- summary-first structure

---

# Public Audience

## Characteristics

- diverse backgrounds
- unstable attention
- emotionally influenced cognition

## Design Strategy

Use:

- storytelling
- visual metaphors
- emotional resonance
- simplified explanations

Reduce:

- technical jargon
- excessive specialization

---

- presentation rhythm
- cognitive density

- visual complexity
- narrative strategy
