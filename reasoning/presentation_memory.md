# Presentation Memory System

This system defines how AI presentation intelligence should maintain long-term presentation preferences.

Presentation quality improves when the system remembers:

- user preferences
- audience preferences
- visual tendencies
- narrative tendencies
- cognitive expectations

Presentation intelligence should evolve over time.

---

# Core Principle

Presentation systems should adapt to:

recurring user cognition patterns.

The system should remember what consistently improves communication quality.

---

# Memory Categories

## 1. Visual Preference Memory

Remember recurring preferences such as:

- preferred color systems
- whitespace preference
- visual density preference
- diagram preference
- hierarchy preference
- layout preference

### Example

User repeatedly prefers:

- blue professional tones
- clean layouts
- restrained decoration
- mechanism-based visuals

The system should reinforce these tendencies.

---

## 2. Narrative Preference Memory

Remember recurring narrative patterns such as:

- progressive storytelling
- mechanism-first explanation
- teaching-oriented pacing
- concise section transitions
- interdisciplinary framing

Narrative rhythm should adapt to user tendencies.

---

## 3. Cognitive Preference Memory

Remember recurring cognitive patterns such as:

- low-density preference
- simplified explanation preference
- high readability preference
- audience-first communication style

The system should preserve cognitive consistency.

---

## 4. Audience Preference Memory

Remember recurring audience targets such as:

- undergraduate students
- interdisciplinary committees
- academic experts
- teaching interview panels

Audience adaptation should improve over time.

---

## 5. Risk-control Preference Memory

Remember recurring concerns such as:

- reducing questioning risk
- avoiding over-claiming expertise
- reducing cognitive overload
- avoiding visual clutter

The system should proactively manage known risks.

---

# Adaptive Memory Rules

The system should:

- reinforce successful presentation patterns
- preserve stable communication styles
- reduce repeated user corrections
- improve future presentation alignment

Memory should improve presentation efficiency over time.

---

# Memory Failure Signals

The system fails if:

- repeated user corrections occur
- recurring preferences are ignored
- presentation style becomes inconsistent
- cognitive expectations are violated

---

# Long-term Intelligence Principle

Presentation intelligence improves through:

continuous adaptation
to recurring communication patterns.

---

# Final Principle

A presentation system becomes intelligent when:

it remembers how the user communicates best.
