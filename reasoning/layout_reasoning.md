# Layout Reasoning Engine

Presentation layout is not decoration.

Layout controls:

- audience attention
- reading sequence
- cognitive rhythm
- semantic emphasis
- narrative understanding

AI must select layouts intentionally.

---

# Core Principle

A slide layout should answer:

- What should audience see first?
- What should audience understand next?
- What information is primary?
- What information is secondary?

Layout is an attention-guidance system.

---

# Center-focus Layout

- one dominant visual center
- strong visual emphasis
- minimal distraction

## Best For

- key concepts
- cover slides
- important conclusions
- single-message slides

## Cognitive Effect

Creates:

- focus
- memorability
- clarity

# Hierarchy-driven Layout

Slides should establish clear hierarchy.

Recommended hierarchy:

1. title
2. main visual
3. key message
4. supporting information
5. details

Avoid equal visual weight everywhere.

---

# Attention Routing Principle

Audience attention should move intentionally.

Recommended reading paths:

- Z-pattern
- F-pattern
- center-focus
- top-down

AI should guide eye movement naturally.

---

# Layout Simplification

Good layouts reduce cognitive navigation difficulty.

Reduce:

- excessive segmentation
- irregular positioning
- random spacing
- visual confusion

Prefer:

- alignment consistency
- visual rhythm
- spacing stability
- semantic grouping

---

# Adaptive Layout Selection

AI should select layouts based on:

- presentation scenario
- audience type
- information density
- narrative structure
- visual semantics

Layout should emerge from cognition,
not decoration.
