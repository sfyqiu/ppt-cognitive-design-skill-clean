# Rendering Diagnostics Artifact System

## Purpose

This document defines the rendering diagnostics artifact used by the AI-native presentation cognition system.

The objective is NOT:
technical renderer debugging.

The objective IS:
communication cognition diagnosis.

Traditional rendering diagnostics focus on:

- rendering failures
- coordinate conflicts
- visual overflow
- object collisions

This system diagnoses:

- hierarchy instability
- cognition overload
- mechanism confusion
- audience accessibility problems
- semantic inconsistency
- communication breakdowns

The rendering diagnostics artifact exists to explain:

```text
Why communication quality failed.
```

NOT only:

```text
Why rendering technically failed.
```

---

# Core Rendering Diagnostics Philosophy

Diagnostics are NOT:

engineering logs.

Diagnostics ARE:

communication cognition analysis.

The diagnostics system should answer:

```text
What communication behavior failed?
↓
Why did audience understanding weaken?
↓
Which cognition system caused instability?
↓
How should refinement proceed?
```

Diagnostics should preserve:

refinement precision.
