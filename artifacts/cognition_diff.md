# Cognition Diff Artifact System

## Purpose

This document defines the cognition diff artifact used by the AI-native presentation cognition system.

The objective is NOT:
visual change tracking.

The objective IS:
cognition-level refinement tracking.

Traditional revision systems compare:

- pixels
- coordinates
- object changes
- styling differences

This system compares:

- hierarchy behavior
- communication logic
- mechanism readability
- audience accessibility
- semantic stability
- cognition effectiveness

The cognition diff exists to explain:

```text
How communication cognition changed.
```

NOT merely:

```text
What visual objects changed.
```

---

# Core Cognition Diff Philosophy

A cognition diff is NOT:

a rendering comparison.

A cognition diff IS:

a communication evolution trace.

The diff should answer:

```text
What cognition problem existed?
↓
What communication behavior changed?
↓
How did understanding improve?
↓
What remained intentionally stable?
```

The cognition diff should preserve:

refinement explainability.

---

# Global Cognition Diff Flow

The cognition diff coordinates:

```text
Previous Cognition State
↓
Refinement Patch
↓
Updated Cognition State
↓
Behavior Comparison
↓
Communication Impact Analysis
↓
Stability Validation
↓
Diff Generation
```

The cognition diff becomes:

the cognition evolution artifact.

# Cognition Diff Rules

The cognition diff must:

- track cognition evolution
- explain communication improvements
- preserve presentation continuity
- stabilize hierarchy understanding
- validate refinement precision
- preserve semantic consistency
- support persistent collaboration

Diffs should prioritize:

communication explainability.

---

# Anti-AI-Slop Diff Rules

The cognition diff must actively prevent:

- pixel-level comparison thinking
- coordinate-only tracking
- decorative refinement reporting
- regeneration-oriented revision behavior
- presentation identity instability
- refinement fragmentation

Diffs should feel:

editorial,
intentional,
and cognition-aware.

---

# Final Validation

Before diff approval:

Check:

- Is the cognition problem clearly explained?
- Is the communication improvement visible?
- Is presentation identity preserved?
- Is hierarchy evolution understandable?
- Is semantic continuity stable?
- Is refinement localized?
- Does the diff feel cognition-oriented rather than visually technical?

If NO:

refine cognition diff behavior.
