# Iterative Presentation Refinement System

## Purpose

This document defines
how the AI presentation cognition system
should support
continuous iterative collaboration
between user and agent.

The objective is NOT:
repeated regeneration.

The objective IS:
stable communication evolution
through controlled cognitive refinement.

The system should continuously improve:

- hierarchy clarity
- communication effectiveness
- audience interpretability
- professional stability
- teaching accessibility
- mechanism readability

while preserving:

- presentation identity
- visual atmosphere
- narrative continuity
- audience adaptation
- cognition stability

The refinement process should feel:

human-guided,
intentional,
and progressively stabilized.

---

# Core Refinement Philosophy

Presentation refinement is NOT:

generate
→ replace
→ regenerate.

It IS:

analyze
→ diagnose
→ refine
→ stabilize
→ improve.

The system must preserve:

existing successful communication structures.

Only unstable communication zones
should change.

The objective is:

maximum communication improvement
with minimum disruption.

---

# Global Refinement Loop

The system executes:

Initial Generation
↓
User Review
↓
Feedback Interpretation
↓
Cognitive Diagnosis
↓
Local Refinement Planning
↓
Hierarchy Stabilization
↓
Evaluation
↓
Iterative Improvement
↓
Final Stabilization

Each iteration should improve:

communication precision.

---

# Stage 1 — Initial Generation Stabilization

The first generated presentation establishes:

- workflow identity
- dominant skill
- visual atmosphere
- hierarchy logic
- audience adaptation
- semantic color behavior
- spacing rhythm

These systems become:

persistent cognition anchors.

Future refinement must preserve:

presentation personality continuity.

---

# Stage 2 — User Feedback Interpretation

The system must interpret:

what communication problem exists.

User feedback may be indirect.

Examples:

“Too dense”
→ cognitive overload
→ annotation saturation
→ weak whitespace hierarchy

“Looks amateur”
→ unstable typography
→ poor spacing rhythm
→ weak editorial hierarchy

“Too difficult for students”
→ excessive abstraction
→ terminology overload
→ weak visual explanation

The system must diagnose:

root cognitive problems,
NOT literal wording.

---

# Stage 3 — Communication Failure Diagnosis

The system must identify:

which communication layer failed.

Possible failures include:

- hierarchy instability
- weak mechanisms
- visual competition
- narrative fragmentation
- audience mismatch
- teaching inaccessibility
- professionalism weakness
- excessive density
- visual noise

Diagnosis should determine:

smallest effective correction path.

---

# Stage 4 — Cognition Memory Preservation

The system must preserve:

- active workflow
- dominant skill
- active style
- visual atmosphere
- hierarchy systems
- audience adaptation
- semantic color logic

The system must NEVER:

accidentally restart
presentation cognition behavior.
